I. Object là gì?

Định nghĩa

Object là một kiểu dữ liệu trong JavaScript cho phép lưu trữ nhiều giá trị dưới dạng cặp key-value (thuộc tính: giá trị).

Cấu trúc Object

name
"Nguyễn Văn A" string
age
25 number
isStudent
true boolean
greet
function() { ... } function

Mục đích sử dụng

  • Gom nhóm thông tin: Lưu trữ các thuộc tính liên quan về một đối tượng thực tế
  • Quản lý cấu trúc phức tạp: Dễ dàng truy cập và thao tác với dữ liệu có cấu trúc
  • Lập trình hướng đối tượng: Nền tảng cho OOP trong JavaScript
  • Tổ chức code: Phân nhóm các hàm và biến liên quan

Cú pháp cơ bản

Cú pháp khai báo object
const objectName = {
  key1: value1,
  key2: value2,
  // ...
};
Ví dụ thực tế:
person.js
const person = {
  name: "Nguyễn Văn Nam",
  age: 25,
  isStudent: true,
  address: {
    city: "Hà Nội",
    district: "Cầu Giấy"
  },
  skills: ["JavaScript", "HTML", "CSS"]
};
Lưu ý quan trọng
  • Key (tên thuộc tính) phải là chuỗi, nếu không sẽ được tự động ép kiểu thành chuỗi
  • Value có thể là bất kỳ kiểu dữ liệu nào: chuỗi, số, boolean, array, object khác, hoặc function
  • Các cặp key-value được phân cách bằng dấu phẩy (,)
  • Tên object nên có ý nghĩa, phản ánh đúng mục đích sử dụng

II. Cách tạo Object

1. Object Literal

Phổ biến nhất, tạo object bằng dấu {}

const person = {
  name: "Nam",
  age: 25
};
Ưu điểm: Ngắn gọn, dễ đọc, hiệu suất tốt

2. new Object()

Sử dụng constructor Object()

const car = new Object();
car.brand = "Toyota";
car.color = "Red";
Khi dùng: Khi cần thêm thuộc tính từ từ

3. Constructor Function

Tạo nhiều object cùng cấu trúc

function Car(brand, color) {
  this.brand = brand;
  this.color = color;
}
const myCar = new Car("Toyota", "Red");

4. Class (ES6+)

Cách hiện đại, hỗ trợ OOP đầy đủ

class Student {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}
const s1 = new Student("Quang", 20);

III. Thuộc tính & Phương thức

1. Thuộc tính (Property)

Thuộc tính là cặp key:value lưu thông tin của object. Có 2 cách truy cập:

Dot Notation

Dùng dấu chấm .

const user = {
  name: "Hưng",
  age: 28
};

console.log(user.name); // "Hưng"
user.age = 29; // Cập nhật
Dùng khi: Key là tên hợp lệ (không có ký tự đặc biệt, không bắt đầu bằng số)

Bracket Notation

Dùng dấu ngoặc []

const user = {
  "full-name": "Nguyễn Văn A",
  age: 28
};

console.log(user["full-name"]);
const key = "age";
console.log(user[key]); // 28
Bắt buộc dùng khi: Key có ký tự đặc biệt, khoảng trắng, hoặc là biến động

2. Thêm/Sửa/Xóa thuộc tính

Thao tác với thuộc tính
const product = {
  name: "Laptop"
};

// Thêm thuộc tính
product.price = 1500;
product["brand"] = "Dell";

// Sửa thuộc tính
product.name = "Gaming Laptop";

// Xóa thuộc tính
delete product.price;
delete product["brand"];

console.log(product); // { name: "Gaming Laptop" }

3. Phương thức (Method)

Phương thức là thuộc tính có giá trị là hàm. Dùng để thực hiện hành vi của object.

Khai báo thông thường

const dog = {
  name: "Lu",
  speak: function() {
    console.log(`${this.name} nói: Gâu Gâu!`);
  }
};

Khai báo rút gọn (ES6)

const cat = {
  name: "Miu",
  speak() {
    console.log(`${this.name} kêu: Meo Meo!`);
  }
};

IV. Duyệt Object

Duyệt object là quá trình truy cập lần lượt từng cặp key:value trong object.

Các phương pháp duyệt

for...in (Truyền thống)

const student = {
  name: "Hải",
  age: 23,
  score: 8.5
};

for (let key in student) {
  console.log(`${key}: ${student[key]}`);
}
Lưu ý: Duyệt cả thuộc tính kế thừa. Dùng hasOwnProperty() để chỉ lấy thuộc tính riêng.

Object.entries() (Hiện đại)

const student = {
  name: "Hải",
  age: 23,
  score: 8.5
};

for (let [key, value] of Object.entries(student)) {
  console.log(`${key} => ${value}`);
}
Ưu điểm: Chỉ duyệt thuộc tính riêng, dễ đọc, hỗ trợ destructuring

Các phương thức hữu ích

Object methods
const person = {
  name: "An",
  age: 25,
  job: "Developer"
};

// Lấy tất cả keys
const keys = Object.keys(person);
console.log(keys); // ["name", "age", "job"]

// Lấy tất cả values
const values = Object.values(person);
console.log(values); // ["An", 25, "Developer"]

// Lấy cả key và value
const entries = Object.entries(person);
console.log(entries); // [["name", "An"], ["age", 25], ["job", "Developer"]]

V. Destructuring Object

Destructuring là cú pháp giúp trích xuất dữ liệu từ object ra biến một cách nhanh chóng và ngắn gọn.

Cú pháp cơ bản

Destructuring cơ bản
const product = {
  name: "Mouse",
  price: 200,
  brand: "Logitech",
  category: "Electronics"
};

// Destructuring cơ bản
const { name, price } = product;
console.log(name); // "Mouse"
console.log(price); // 200

// Đổi tên biến
const { brand: productBrand } = product;
console.log(productBrand); // "Logitech"

// Giá trị mặc định
const { warranty = "12 months" } = product;
console.log(warranty); // "12 months"

Ứng dụng nâng cao

Destructuring nâng cao
const user = {
  name: "Minh",
  age: 30,
  contact: {
    email: "minh@example.com",
    phone: "0123456789"
  },
  scores: [8, 9, 7]
};

// Destructuring lồng nhau
const {
  name,
  contact: { email },
  scores: [firstScore]
} = user;

console.log(name); // "Minh"
console.log(email); // "minh@example.com"
console.log(firstScore); // 8

VI. Copy Object

Vấn đề: Copy tham chiếu

Khi gán object cho biến mới, thực chất chỉ copy tham chiếu, không copy giá trị!

Vấn đề copy tham chiếu
const original = {
  name: "An",
  address: {
    city: "Hanoi"
  }
};

// Copy tham chiếu (SAI!)
const wrongCopy = original;
wrongCopy.name = "Bình";
wrongCopy.address.city = "Saigon";

console.log(original.name); // "Bình" ❌ Bị thay đổi!
console.log(original.address.city); // "Saigon" ❌ Bị thay đổi!

Các cách copy object

Shallow Copy

Sao chép nông - chỉ copy cấp 1

// Spread operator (ES6)
const copy1 = { ...original };

// Object.assign()
const copy2 = Object.assign({}, original);
Hạn chế: Object lồng bên trong vẫn bị tham chiếu

Deep Copy

Sao chép sâu - copy toàn bộ

// JSON (phổ biến)
const deepCopy = JSON.parse(JSON.stringify(original));

// Lodash library
// const deepCopy = _.cloneDeep(original);
Lưu ý: JSON không copy được function, undefined, Symbol, Date

So sánh Shallow vs Deep Copy

Demo Shallow vs Deep Copy
const person = {
  name: "An",
  address: {
    city: "Hanoi"
  }
};

// Shallow copy
const shallowCopy = { ...person };
shallowCopy.name = "Bình";
shallowCopy.address.city = "Saigon";

console.log(person.name); // "An" ✅ Không đổi
console.log(person.address.city); // "Saigon" ❌ Bị đổi!

// Deep copy
const deepCopy = JSON.parse(JSON.stringify(person));
deepCopy.address.city = "Hue";

console.log(person.address.city); // "Saigon" ✅ Không đổi

VII. Tính năng nâng cao

1. Optional Chaining (?.)

Giúp truy cập thuộc tính sâu mà không gây lỗi nếu giá trị trung gian là undefined hoặc null.

Optional Chaining
const user = {
  profile: {
    name: "Mai"
  }
};

// Truy cập thông thường (có thể lỗi)
// const email = user.contact.email; // Lỗi!

// Với Optional Chaining
const email = user.contact?.email;
console.log(email); // undefined (không lỗi)

const city = user.profile?.address?.city;
console.log(city); // undefined (không lỗi)

const name = user.profile?.name ?? "Guest";
console.log(name); // "Mai"

2. Computed Property Names

Tạo key từ giá trị biến động hoặc biểu thức.

Computed Property
const prop = "age";
const user = {
  name: "Hoàng",
  [prop]: 27,
  ["user_" + prop]: "user_age",
  ["full" + "Name"]: "Nguyễn Văn Hoàng"
};

console.log(user.age); // 27
console.log(user.user_age); // "user_age"
console.log(user.fullName); // "Nguyễn Văn Hoàng"

3. Shorthand Property & Method

Viết tắt khi tên biến trùng với tên key.

Shorthand Syntax
const name = "Lan";
const age = 25;
const price = 1000;

// Khai báo thông thường
// const person = { name: name, age: age };

// Shorthand (ES6)
const person = { name, age }; // Tương đương { name: name, age: age }

const product = {
  name: "TV",
  price, // Tương đương price: price
  showInfo() { // Method shorthand
    console.log(`${this.name} - $${this.price}`);
  }
};

product.showInfo(); // "TV - 1000"
