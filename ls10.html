1. SỰ KIỆN CLICK

Định nghĩa:

Kích hoạt khi người dùng nhấn chuột vào một phần tử HTML (thường là nút).

Cú pháp:

Cú pháp
element.addEventListener("click", function(event) {
    // xử lý khi click
});

Ví dụ:

HTML
<button id="btn">Nhấn tôi</button>
<p id="info"></p>
JS
document.getElementById("btn").addEventListener("click", function() {
    document.getElementById("info").innerText = "Bạn vừa click!";
});

Kết quả: Khi click nút, dòng "Bạn vừa click!" sẽ hiển thị.

Thử nghiệm sự kiện click

2. SỰ KIỆN INPUT

Định nghĩa:

Xảy ra khi người dùng nhập dữ liệu vào <input>, <textarea>,...

Cú pháp:

Cú pháp
inputElement.addEventListener("input", function(e) {
    let value = e.target.value;
});

Ví dụ:

HTML
<input type="text" id="txtName" placeholder="Nhập tên">
<p id="preview"></p>
JS
document.getElementById("txtName").addEventListener("input", function(e) {
    document.getElementById("preview").innerText = "Tên: " + e.target.value;
});

Kết quả: Gõ tới đâu, kết quả hiển thị tới đó.

Thử nghiệm sự kiện input

3. SỰ KIỆN SUBMIT VÀ VALIDATE FORM

Định nghĩa:

Sự kiện submit xảy ra khi gửi form (bấm nút submit).

Cú pháp:

Cú pháp
formElement.addEventListener("submit", function(e) {

KHÔNG ĐƯỢC ĐỂ TRỐNG (REQUIRED FIELD VALIDATION)

Mục đích:

Đảm bảo người dùng không để trống ô input.

Ví dụ:

HTML
<input type="text" id="username" placeholder="Nhập tên">
JS
let name = document.getElementById("username").value.trim();

if (name === "") {
    alert("Tên không được để trống");
}

Giải thích:

.value lấy giá trị người dùng nhập.
.trim() loại bỏ khoảng trắng ở đầu/cuối (vd: " " cũng bị coi là rỗng).
Nếu rỗng → báo lỗi.

KIỂM TRA ĐỘ DÀI CHUỖI (LENGTH VALIDATION)

Mục đích:

Xác định xem người dùng có nhập đủ độ dài tối thiểu hay tối đa không.

Ví dụ:

HTML
<input type="text" id="username">
JS
let name = document.getElementById("username").value.trim();

if (name.length < 5) {
    alert("Tên phải có ít nhất 5 ký tự");
}

Giải thích:

name.length là số lượng ký tự trong chuỗi.
So sánh length để kiểm tra độ dài.

KIỂM TRA EMAIL ĐÚNG ĐỊNH DẠNG (EMAIL FORMAT VALIDATION)

Mục đích:

Xác minh rằng email nhập đúng chuẩn (@, .com, v.v.).
Sử dụng regex để check.

REGEX LÀ GÌ?

Định nghĩa:

Regex (viết tắt của Regular Expression) là biểu thức chính quy – một chuỗi ký tự đặc biệt dùng để kiểm tra, tìm kiếm hoặc thay thế chuỗi khác theo một khuôn mẫu (pattern).

Regex dùng để làm gì?

Công dụng thực tế Ví dụ
Kiểm tra định dạng email email@gmail.com
Kiểm tra số điện thoại 0987654321
Kiểm tra mật khẩu mạnh Abc@1234

CÚ PHÁP CƠ BẢN CỦA REGEX

Ký hiệu Ý nghĩa Ví dụ
. bất kỳ ký tự nào (trừ xuống dòng) a.c khớp abc, axc
\d chữ số (0–9) \d\d khớp 12
\w chữ, số, gạch dưới \w+ khớp abc123
\s khoảng trắng a\sb khớp a b
^ bắt đầu chuỗi ^Hello khớp Hello world
$ kết thúc chuỗi world$ khớp hello world
[abc] 1 trong các ký tự a, b, c [aeiou] khớp nguyên âm
[^abc] KHÔNG phải a, b, c [^0-9] không phải số
a{3} a lặp đúng 3 lần a{3} khớp aaa
a+ a lặp ≥1 lần a+ khớp a, aaa
a* a lặp 0 hoặc nhiều lần a* khớp ``, a, aa
(abc) nhóm (ab)+ khớp ab, abab
| hoặc a|b khớp a hoặc b

Ví dụ:

Kiểm tra Email
regex = /^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$/;
// ^ bắt đầu
// [\w-\.]+ là phần tên trước @ (chữ + số + dấu - .)
// @ là bắt buộc
// ([\w-]+\.)+ là tên miền (vd: gmail.)
// [\w-]{2,4} là đuôi (com, net,...)
// $ kết thúc
Số điện thoại Việt Nam
regex = /^0\d{9}$/
// ^0 bắt đầu bằng số 0
// \d{9}: theo sau là 9 số
// $ kết thúc
Mật khẩu mạnh
regex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&]).{8,}$/
// (?=.*[a-z]): ít nhất 1 chữ thường
// (?=.*[A-Z]): ít nhất 1 chữ hoa
// (?=.*\d): ít nhất 1 số
// (?=.*[@$!%*?&]): ít nhất 1 ký tự đặc biệt
// .{8,}: ít nhất 8 ký tự

Kiểm tra Regex trong JS:

Sử dụng hàm: pattern.test(stringTest)

Ví dụ
<input type="email" id="email">

let email = document.getElementById("email").value.trim();

let regex = /^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$/;

if (!regex.test(email)) {
    alert("Email không hợp lệ");
}

Giải thích:

regex là biểu thức chính quy để kiểm tra định dạng email.
test() kiểm tra xem chuỗi có khớp không.

MẬT KHẨU PHẢI ĐỦ MẠNH (STRONG PASSWORD VALIDATION)

Mục đích:

Đảm bảo mật khẩu đủ an toàn: ít nhất 8 ký tự, gồm chữ hoa, chữ thường, số, ký tự đặc biệt.

Ví dụ:

HTML
<input type="password" id="password">
JS
let password = document.getElementById("password").value;

let strongRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&]).{8,}$/;

if (!strongRegex.test(password)) {
    alert("Mật khẩu yếu! Phải có ít nhất 8 ký tự, bao gồm chữ hoa, thường, số, ký tự đặc biệt.");
}

Giải thích:

(?=.*[a-z]): ít nhất 1 chữ thường
(?=.*[A-Z]): ít nhất 1 chữ hoa
(?=.*\d): ít nhất 1 số
(?=.*[@$!%*?&]): ít nhất 1 ký tự đặc biệt
.{8,}: ít nhất 8 ký tự

SO KHỚP MẬT KHẨU (CONFIRM PASSWORD)

HTML
<input type="password" id="password">
<input type="password" id="confirmPassword">
JS
let pass1 = document.getElementById("password").value;
let pass2 = document.getElementById("confirmPassword").value;

if (pass1 !== pass2) {
    alert("Mật khẩu không khớp");
}

VÍ DỤ ĐẦY ĐỦ: FORM ĐĂNG KÝ

HTML
<h3>Form Đăng Ký</h3>
<form id="registerForm">
  <input type="text" id="username" placeholder="Tên đăng nhập"><br>
  <input type="email" id="email" placeholder="Email"><br>
  <input type="password" id="password" placeholder="Mật khẩu"><br>
  <button type="submit">Đăng ký</button>
</form>
<p id="error" style="color:red;"></p>
JS
document.getElementById("registerForm").addEventListener("submit", function(e) {
    e.preventDefault(); // chặn reload
    let username = document.getElementById("username").value.trim();
    let email = document.getElementById("email").value.trim();
    let password = document.getElementById("password").value.trim();
    let error = "";

    // Kiểm tra tên đăng nhập
    if (username === "") {
        error += "Tên đăng nhập không được để trống.<br>";
    } else if (username.length < 3) {
        error += "Tên đăng nhập phải có ít nhất 3 ký tự.<br>";
    }

    // Kiểm tra email
    const emailPattern = /^[^ ]+@[^ ]+\.[a-z]{2,3}$/;
    if (!email.match(emailPattern)) {
        error += "Email không hợp lệ.<br>";
    }

    // Kiểm tra mật khẩu
    if (password.length < 6) {
        error += "Mật khẩu phải có ít nhất 6 ký tự.<br>";
    }

    // Hiển thị lỗi hoặc thành công
    if (error !== "") {
        document.getElementById("error").innerHTML = error;
    } else {
        document.getElementById("error").style.color = "green";
        document.getElementById("error").innerHTML = "Đăng ký thành công!";
        // có thể gửi dữ liệu lên server tại đây
    }
});

Kết quả:
Khi người dùng bấm "Đăng ký":
Nếu thiếu thông tin → hiện lỗi ngay dưới form
Nếu đúng → hiện "Đăng ký thành công!"

9. CHUYỂN TRANG DETAIL

Cách đơn giản nhất: Sử dụng thẻ <a> trong HTML

HTML
<a href="detail.html">Xem chi tiết</a>

Cách này chuyển sang trang detail.html ngay khi người dùng click.

Truyền dữ liệu qua URL (dạng query string)

HTML
<a href="detail.html?id=123">Xem chi tiết SP 123</a>

Ở trang detail.html, bạn lấy dữ liệu bằng JavaScript:

JS (trang detail.html)
<script>
  // Lấy tham số "id" trên URL
  const urlParams = new URLSearchParams(window.location.search);
  const id = urlParams.get('id');
  console.log("ID sản phẩm:", id);
</script>

Dùng JavaScript để điều hướng (redirect)

HTML
<button onclick="goToDetail(123)">Xem chi tiết</button>
JS
<script>
  function goToDetail(id) {
    // Chuyển trang và truyền id lên URL
    window.location.href = 'detail.html?id=' + id;
  }
</script>

Nếu dùng danh sách động (JS sinh ra các item):

HTML
<div id="products"></div>
JS
let products = [
  { id: 1, name: "Áo thun" },
  { id: 2, name: "Giày thể thao" }
];

let container = document.getElementById("products");

products.forEach(p => {
  let item = document.createElement("div");
  item.innerHTML = `<a href="detail.html?id=${p.id}">${p.name}</a>`;
  container.appendChild(item);
});

Thử nghiệm chuyển trang detail

Các liên kết sẽ chuyển đến trang detail.html với ID tương ứng. Trong môi trường demo, bạn có thể xem URL thay đổi.
