1. let VÀ const TRONG ES6

let là gì?

  • Dùng để khai báo biến.
  • Có block scope (phạm vi trong { })
  • Có thể gán lại giá trị mới
Ví dụ let
let x = 10;
x = 20; // ok
console.log(x); // 20

const là gì?

  • Dùng để khai báo hằng số (constant)
  • Có block scope
  • Không được gán lại giá trị
Ví dụ const
const y = 5;
y = 10; // Lỗi: Assignment to constant variable

Nhưng với object hoặc array, bạn vẫn có thể thay đổi nội dung bên trong:

Ví dụ const với object/array
const arr = [1, 2];
arr.push(3); // hợp lệ
console.log(arr); // [1, 2, 3]

const obj = { name: "Tom" };
obj.name = "Jerry"; // OK

So sánh var, let, const

Tính năng var let const
Phạm vi (scope) function scope block scope {} block scope {}
Hoisting Có (lên đầu hàm) Có nhưng không dùng trước khi khai báo Có nhưng không dùng trước khi khai báo
Gán lại Có thể Có thể Không thể
Khai báo lại Có thể Không thể Không thể

Hoisting

Ví dụ hoisting
console.log(a); // undefined
var a = 10;

console.log(b); // Lỗi ReferenceError
let b = 20;

Giải thích:

var được "kéo lên" đầu nhưng chưa có giá trị
let/const cũng được hoisting nhưng nằm trong "Temporal Dead Zone" → không thể sử dụng trước khi khai báo

3. TEMPLATE STRING (CHUỖI MẪU)

Dùng dấu (backtick) thay vì dấu nháy đơn ' hoặc nháy kép "

Ví dụ cơ bản
const name = "Alice";
const message = `Hello, ${name}!`;
console.log(message); // Hello, Alice!

Lưu ý: Dấu backtick nằm ngay dưới phím Esc trên bàn phím tiếng Anh.

Chèn biến hoặc biểu thức bằng ${...}

Ví dụ
let a = 5;
let b = 10;

console.log(`Tổng của ${a} và ${b} là ${a + b}`);

Chuỗi nhiều dòng

Trước ES6:

Trước ES6
let msg = "Hello\n" +
          "This is line 2\n" +
          "This is line 3";

Với ES6:

ES6
let msg = `Hello
This is line 2
This is line 3`;

Sử dụng trong HTML

Bạn có thể dùng template string để tạo HTML động:

Ví dụ
const product = "Laptop";

const html = `

<p>Price: ${price.toLocaleString()} VND</p>

document.body.innerHTML = html;

4. DESTRUCTURING – PHÂN RÃ MẢNG & ĐỐI TƯỢNG

Array Destructuring (Phân rã Mảng)

Cú pháp:

Ví dụ
const arr = [1, 2, 3];
const [a, b, c] = arr;

console.log(a); // 1
console.log(b); // 2

Bỏ qua phần tử:

Ví dụ
const [x, , z] = [10, 20, 30];
console.log(z); // 30

Giá trị mặc định:

Ví dụ
const [m, n, o = 100] = [1, 2];
console.log(o); // 100

Trao đổi biến (swap):

Ví dụ
let a = 1, b = 2;
[a, b] = [b, a];
console.log(a, b); // 2 1

Object Destructuring (Phân rã Đối tượng)

Ví dụ
const user = { name: "Alice", age: 25 };
const { name, age } = user;
console.log(name); // Alice

Đổi tên biến:

Ví dụ
const person = { firstName: "Bob", age: 30 };
const { firstName: ten, age: tuoi } = person;
console.log(ten); // Bob

Giá trị mặc định:

Ví dụ
const { role = "guest" } = {};
console.log(role); // guest

Lồng nhau (nested):

Ví dụ
const obj = {
  user: {
    name: "Tom",
    address: {
      city: "Hanoi"
    }
  }
};

const {
  user: {
    name,
    address: { city }
  }
} = obj;

console.log(name); // Tom
console.log(city); // Hanoi

Kết hợp với Rest operator

Vị trí dùng Ý nghĩa
[a, ...b] = arr b chứa các phần tử còn lại
{x, ...y} = obj y chứa thuộc tính còn lại
Ví dụ
const [a, ...rest] = [1, 2, 3, 4];
console.log(rest); // [2, 3, 4]

const { x, ...others } = { x: 1, y: 2, z: 3 };
console.log(others); // { y: 2, z: 3 }

5. REST OPERATOR

Mục đích:

Rest operator (...) dùng để gom nhiều phần tử (của mảng, đối số hàm, hoặc thuộc tính object) thành một mảng duy nhất hoặc một object duy nhất.

Hữu ích khi:

  • Muốn nhận nhiều đối số mà không cần liệt kê rõ ràng từng cái.
  • Sao chép phần còn lại của mảng/object sau khi đã "tách" một vài phần tử đầu.

Với mảng:

Nối mảng

Ví dụ
const arr1 = [1, 2];
const arr2 = [3, 4];

const combined = [...arr1, ...arr2];
console.log(combined); // [1, 2, 3, 4]

Sao chép mảng (deep copy nông)

Ví dụ
const original = [10, 20];
const copy = [...original];

copy.push(30);
console.log(original); // [10, 20]
console.log(copy); // [10, 20, 30]

Truyền mảng vào hàm

Ví dụ
const numbers = [5, 10, 15];

console.log(Math.max(...numbers)); // 15

Với object:

Sao chép đối tượng

Ví dụ
const person = { name: "Linh", age: 22 };
const clone = { ...person };

console.log(clone); // { name: "Linh", age: 22 }

Gộp nhiều object

Ví dụ
const a = { x: 1 };
const b = { y: 2 };

const merged = { ...a, ...b };
console.log(merged); // { x: 1, y: 2 }

Ghi đè thuộc tính

Ví dụ
const obj1 = { name: "Nam", age: 18 };
const obj2 = { age: 20 };

const final = { ...obj1, ...obj2 };
console.log(final); // { name: "Nam", age: 20 }

So sánh ...rest và ...spread

Loại Vị trí Ý nghĩa
...rest Trong destructuring hoặc tham số Gom lại phần còn lại
...spread Trong biểu thức hoặc khởi tạo Trải ra các phần tử
Ví dụ minh họa
const arr = [1, 2, 3, 4];
const [first, ...rest] = arr; // rest = [2, 3, 4]

const newArr = [...rest, 5, 6]; // newArr = [2, 3, 4, 5, 6]

Thử nghiệm Rest Operator

6. MODULE (IMPORT/EXPORT)

Mục đích:

  • Giúp chia nhỏ mã nguồn thành các module riêng biệt, dễ quản lý và tái sử dụng.
  • Hỗ trợ import/export trực tiếp trong JavaScript ES6.
  • Tránh biến toàn cục và giúp bảo mật code tốt hơn.

Lưu ý: Khi dùng import/export, cần chạy trong môi trường hỗ trợ module như:

  • Trình duyệt (dùng <script type="module">)
  • Node.js (dùng file .mjs hoặc bật "type": "module" trong package.json)

Ví dụ:

📁 File: mathUtils.js

mathUtils.js
// Export biến
export const PI = 3.14;

// Export hàm
export function add(a, b) {
  return a + b;
}

// Export mặc định
export default function multiply(a, b) {
  return a * b;
}

📁 File: main.js

main.js
// Import có tên
import { PI, add } from "./mathUtils.js";

// Import mặc định
import multiply from "./mathUtils.js";

console.log(PI); // 3.14
console.log(add(2, 3)); // 5
console.log(multiply(2, 3)); // 6

Giải thích chi tiết:

  • export:
    • export const PI = 3.14;: cho phép file khác import biến này.
    • export function add(...) {}: export hàm có tên.
    • export default: mỗi file chỉ có 1 export mặc định.
  • import:
    • { PI, add }: dùng dấu ngoặc để import các phần tử có tên.
    • import multiply from ...: dùng không có dấu ngoặc cho export mặc định.

Trong HTML:

HTML
<script type="module" src="main.js"></script>

Mặc định module chạy trong chế độ strict (use strict).

7. FUNCTION ARRAY

forEach()

Mục đích: Duyệt qua từng phần tử trong mảng, không trả về giá trị mới (chỉ dùng để thao tác từng phần tử).

Ví dụ
const numbers = [1, 2, 3, 4];

numbers.forEach((num)=> {
  console.log("Số:", num);
});

Giải thích: forEach() lặp qua từng phần tử num trong mảng numbers. Hàm truyền vào là một callback function chạy cho mỗi phần tử. Không thể break hay return khỏi forEach() như for thường.

map()

Mục đích: Trả về một mảng mới sau khi xử lý từng phần tử trong mảng cũ.

Ví dụ
const numbers = [1, 2, 3];

const squares = numbers.map((num)=> {
  return num * num;
});

console.log(squares); // [1, 4, 9]

Giải thích: map() tạo ra mảng mới squares bằng cách bình phương từng số trong numbers. map() luôn trả về một mảng có cùng số phần tử với mảng ban đầu.

filter()

Mục đích: Lọc các phần tử thỏa điều kiện và trả về mảng mới.

Ví dụ
const numbers = [5, 12, 8, 130, 44];

const bigNumbers = numbers.filter((num)=>{
  return num > 10;
});

console.log(bigNumbers); // [12, 130, 44]

Giải thích: filter() trả về mảng chứa các phần tử > 10. Nếu điều kiện true → giữ lại phần tử, nếu false → loại bỏ.

find()

Mục đích: Tìm phần tử đầu tiên trong mảng thỏa điều kiện, nếu không có trả về undefined.

Ví dụ
const users = [
  { id: 1, name: "An" },
  { id: 2, name: "Bình" },
  { id: 3, name: "An" }
];

const result = users.find(user=> {
  return user.name === "An";
});

console.log(result); // { id: 1, name: "An" }

Giải thích: find() dừng lại ngay khi tìm thấy phần tử phù hợp. Trả về object { id: 1, name: "An" } là phần tử đầu tiên tên "An".

findIndex()

Mục đích: Tìm vị trí chỉ mục (index) của phần tử đầu tiên thỏa điều kiện.

Ví dụ
let numbers = [5, 10, 15, 20];
let index = numbers.findIndex(num => num > 10);
console.log(index); // 2

Giải thích: Trả về -1 nếu không tìm thấy. Giống find() nhưng trả về vị trí thay vì phần tử.

some()

Mục đích: Kiểm tra ít nhất một phần tử trong mảng thỏa điều kiện.

Ví dụ
const ages = [3, 10, 18, 20];

const hasAdult = ages.some(age => age >= 18);
console.log(hasAdult); // true

Giải thích: Có ít nhất một phần tử >= 18 nên trả về true.

every()

Mục đích: Kiểm tra tất cả phần tử trong mảng thỏa điều kiện.

Ví dụ
const result = ages.every(age => age >= 3);
console.log(result); // true

Giải thích: Tất cả tuổi >= 3 nên trả về true. Nếu chỉ cần 1 cái sai → false.

sort()

Mục đích: Sắp xếp các phần tử trong mảng. Mặc định là theo chuỗi Unicode.

Ví dụ
let products = [
  { id: 1, name: "iPhone", price: 27000000 },
  { id: 2, name: "Samsung", price: 20000000 },
  { id: 3, name: "Xiaomi", price: 15000000 },
];

// Sắp xếp tăng dần
products.sort((a, b) => a.price - b.price);
console.log(products);

// sắp xếp giảm dần
products.sort((a, b) => b.price - a.price);
console.log(products);

Giải thích: Dễ gây lỗi với số nếu không truyền hàm so sánh (vì mặc định sắp xếp chuỗi). Sửa bằng sort((a, b) => a - b) cho tăng dần, b - a cho giảm dần.

reduce()

Mục đích: Dùng để tính toán tích lũy (tổng, tích, nối chuỗi, gộp đối tượng,...)

Ví dụ: Tính tổng các số
let nums = [1, 2, 3, 4];
let sum = nums.reduce((acc, curr) => acc + curr, 0);
console.log(sum); // 10

Giải thích: acc: giá trị tích lũy (accumulator), curr: phần tử hiện tại, 0: giá trị khởi đầu của acc. Có thể dùng cho các bài toán như: tổng, trung bình, đếm, gộp...

BÀI 1: QUẢN LÝ DANH SÁCH SẢN PHẨM

Cho mảng sản phẩm:

products.js
let products = [
  { id: 1, name: "iPhone 14", price: 27000000, category: "Smartphone" },
  { id: 2, name: "Samsung S22", price: 23000000, category: "Smartphone" },
  { id: 3, name: "MacBook Air", price: 32000000, category: "Laptop" },
  { id: 4, name: "Dell XPS", price: 29000000, category: "Laptop" },
  { id: 5, name: "Sony WH-1000XM5", price: 9000000, category: "Headphone" },
];

Yêu cầu:

1. In ra tên tất cả sản phẩm kèm theo giá (sử dụng forEach)

forEach
products.forEach(p => {
  console.log(`Tên: ${p.name} - Giá: ${p.price.toLocaleString()}đ`);
});

2. Lọc ra các sản phẩm có giá từ 25 triệu trở lên (sử dụng filter)

filter
let expensive = products.filter(p => p.price >= 25000000);
console.log("Sản phẩm đắt tiền:", expensive);

3. Tạo mảng mới chỉ chứa tên sản phẩm (sử dụng map)

map
let productNames = products.map(p => p.name);
console.log("Tên sản phẩm:", productNames);

4. Tính tổng giá tất cả sản phẩm (sử dụng reduce)

reduce
let total = products.reduce((sum, p) => sum + p.price, 0);
console.log("Tổng tiền:", total.toLocaleString() + "đ");

5. Sắp xếp sản phẩm theo giá tăng dần (sử dụng sort)

sort
let sortedAsc = [...products].sort((a, b) => a.price - b.price);
console.log("Sắp xếp tăng:", sortedAsc);

Ghi chú: Dùng [...] để không làm thay đổi mảng gốc.

6. Tìm sản phẩm đầu tiên thuộc loại "Laptop" (sử dụng find)

find
let laptop = products.find(p => p.category === "Laptop");
console.log("Laptop đầu tiên:", laptop);

BTVN: QUẢN LÝ DANH SÁCH SẢN PHẨM

Cho danh sách các sản phẩm như sau:

products.js
let products = [
  { id: "P01", name: "Laptop Dell", category: "Laptop", price: 21000000, quantity: 5 },
  { id: "P02", name: "iPhone 14", category: "Smartphone", price: 19000000, quantity: 10 },
  { id: "P03", name: "Tai nghe Sony", category: "Headphones", price: 2500000, quantity: 20 },
  { id: "P04", name: "Macbook Pro", category: "Laptop", price: 35000000, quantity: 3 },
  { id: "P05", name: "Samsung Galaxy", category: "Smartphone", price: 18000000, quantity: 8 },
];

Yêu cầu xử lý:

  1. In ra danh sách tất cả sản phẩm theo định dạng: "Tên sản phẩm - Danh mục - Giá - Số lượng".
  2. Lọc ra các sản phẩm có giá > 20 triệu.
  3. Tính tổng giá trị kho (giá × số lượng cho từng sản phẩm, rồi cộng lại).
  4. Sắp xếp danh sách sản phẩm theo giá tăng dần.
  5. Tạo mảng mới chỉ chứa tên và giá sản phẩm ([{name, price}]).
  6. Tìm sản phẩm có giá cao nhất.
  7. Đếm số lượng sản phẩm theo từng loại (Laptop, Smartphone...).
