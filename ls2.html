GIỚI THIỆU

1.1 Tổng quan

Trong JavaScript, biến (variables), kiểu dữ liệu (data types) và toán tử (operators) đóng vai trò quan trọng trong việc xây dựng và thực thi mã.

Mục đích học tập:

  • Hiểu và sử dụng biến: Cách khai báo, phạm vi và sử dụng biến trong JavaScript
  • Nắm vững kiểu dữ liệu: Các kiểu dữ liệu cơ bản và phức tạp trong JavaScript
  • Thành thạo toán tử: Sử dụng các toán tử để thao tác với dữ liệu
  • Thực hành ứng dụng: Áp dụng kiến thức vào các ví dụ thực tế

Lưu ý quan trọng

JavaScript là ngôn ngữ lập trình động (dynamically typed), có nghĩa là kiểu dữ liệu của biến được xác định tại thời điểm chạy, không phải tại thời điểm biên dịch. Điều này mang lại sự linh hoạt nhưng cũng đòi hỏi cẩn thận khi làm việc với dữ liệu.

BIẾN (VARIABLES)

1.1.1 Khái niệm và loại biến

Định nghĩa: Biến được dùng để lưu trữ dữ liệu trong chương trình JavaScript.

Các loại biến trong JavaScript:

  • var - Khai báo biến với phạm vi function (cũ)
  • let - Khai báo biến với phạm vi block (ES6+)
  • const - Khai báo hằng số với phạm vi block (ES6+)

1. var - Function Scope

Phạm vi: var có phạm vi hoạt động là function-scope. Điều này có nghĩa là, khi bạn khai báo một biến bằng var bên trong một hàm, bạn không thể truy cập biến đó bên ngoài hàm.

Tái gán: Có thể tái gán giá trị mới cho biến.

Khởi tạo: Khi một biến được khai báo bằng var nhưng chưa được gán giá trị, giá trị của nó là undefined.

Ví dụ sử dụng var
// Khai báo ở mức toàn cục
var globalVar = "I am globally scoped";

function exampleVarFunction() {
    // Khai báo ở mức function-scope
    var localVar = "I am function scoped";
    console.log(globalVar); // Output: "I am globally scoped"
    console.log(localVar); // Output: "I am function scoped"
}

exampleVarFunction();
console.log(globalVar); // Output: "I am globally scoped"
// console.log(localVar); // Lỗi: localVar không được định nghĩa ở mức toàn cục

2. let - Block Scope

Phạm vi: let có phạm vi hoạt động là block-scope. Điều này có nghĩa là, nó chỉ có thể truy cập trong khối mã (bên trong {}) mà nó được khai báo.

Tái gán: Có thể tái gán giá trị mới cho biến.

Khởi tạo: Không tự động khởi tạo. Nếu truy cập biến let trước khi gán giá trị, sẽ nhận được lỗi.

Ví dụ sử dụng let
let globalLet = "I can be accessed anywhere in this script";

if (true) {
    let blockLet = "I am block scoped"; // Khai báo trong một khối mã
    console.log(globalLet); // Output: "I can be accessed anywhere in this script"
    console.log(blockLet); // Output: "I am block scoped"
}

console.log(globalLet); // Output: "I can be accessed anywhere in this script"
// console.log(blockLet); // Lỗi: blockLet không được định nghĩa bên ngoài khối mã của nó

3. const - Hằng số

Phạm vi: Giống như let, const cũng có phạm vi block-scope.

Tái gán: Không thể tái gán giá trị mới cho biến.

Khởi tạo: Phải gán giá trị cho biến khi khai báo.

Ví dụ sử dụng const
const globalConst = "I am a constant value and cannot be changed";

if (true) {
    const blockConst = "I am also block scoped and unchangeable";
    console.log(globalConst); // Output: "I am a constant value and cannot be changed"
    console.log(blockConst); // Output: "I am also block scoped and unchangeable"
}

console.log(globalConst); // Output: "I am a constant value and cannot be changed"
// console.log(blockConst); // Lỗi: blockConst không được định nghĩa bên ngoài khối mã của nó
// globalConst = "Try to change"; // Lỗi: không thể gán giá trị mới cho biến const

Quy tắc đặt tên biến:

  • Tên biến cần bắt đầu bằng một chữ cái, dấu gạch dưới (_) hoặc dấu $
  • Không được bắt đầu bằng số
  • Tên biến phân biệt chữ hoa chữ thường
  • Không sử dụng từ khóa của JavaScript làm tên biến
  • Sử dụng camelCase cho tên biến (ví dụ: firstName, totalAmount)

KIỂU DỮ LIỆU (DATA TYPES)

1.2 Khái niệm và phân loại

Định nghĩa: Kiểu dữ liệu là một khái niệm cơ bản trong lập trình, chỉ ra loại hoặc đặc điểm của một giá trị hoặc biến.

Tại sao cần kiểu dữ liệu?

  • Phân loại giá trị: Giúp xác định và phân loại dữ liệu
  • Hiệu quả bộ nhớ: Mỗi kiểu dữ liệu sử dụng lượng bộ nhớ cụ thể
  • Kiểm tra loại: Đảm bảo tính đúng đắn của chương trình
  • Xác định hành vi: Các phép toán hoạt động khác nhau theo kiểu dữ liệu
  • Hỗ trợ lập trình: Giúp lập trình viên hiểu rõ loại giá trị biến đang lưu trữ

1.2.1 Kiểu dữ liệu nguyên thủy (Primitive Types)

1. Number - Kiểu số

Định nghĩa: Đại diện cho số nguyên hoặc số thực.

Ví dụ cơ bản về Number
let age = 25; // một số nguyên
let average = 5.5; // một số thực
let negative = -10; // số âm
let hex = 0xFF; // số hệ 16 (255)
let octal = 0o377; // số hệ 8 (255)
let binary = 0b11111111; // số hệ 2 (255)
1.1 Phương thức toString()

Mô tả: Chuyển đổi số thành chuỗi

Cú pháp: number.toString([radix])

Ví dụ toString()
let num = 123;
console.log(num.toString()); // "123"
console.log((255).toString(16)); // "ff" - chuyển sang hệ 16
console.log((255).toString(2)); // "11111111" - chuyển sang hệ 2
console.log((255).toString(8)); // "377" - chuyển sang hệ 8
Phân tích:
  • Tham số radix là tùy chọn (2-36), mặc định là 10
  • Trả về chuỗi biểu diễn số trong hệ cơ số được chỉ định
  • Có thể dùng với số thực và số nguyên
1.2 Phương thức toFixed()

Mô tả: Làm tròn số với số chữ số thập phân cố định

Cú pháp: number.toFixed([digits])

Ví dụ toFixed()
let floatNum = 1.23456;
console.log(floatNum.toFixed(2)); // "1.23"
console.log(floatNum.toFixed(0)); // "1"
console.log(floatNum.toFixed(5)); // "1.23456"

let price = 9.99;
console.log("$" + price.toFixed(2)); // "$9.99"
Phân tích:
  • Tham số digits là số chữ số sau dấu thập phân (0-20)
  • Trả về chuỗi, không phải số
  • Tự động làm tròn theo quy tắc làm tròn số
  • Dùng số âm sẽ báo lỗi
1.3 Phương thức toPrecision()

Mô tả: Định dạng số với độ chính xác cụ thể

Cú pháp: number.toPrecision([precision])

Ví dụ toPrecision()
let num = 123.456;
console.log(num.toPrecision(5)); // "123.46"
console.log(num.toPrecision(2)); // "1.2e+2"
console.log(num.toPrecision(7)); // "123.4560"
Phân tích:
  • Tham số precision là số chữ số có nghĩa (1-21)
  • Trả về chuỗi biểu diễn số với độ chính xác được chỉ định
  • Có thể trả về dạng khoa học nếu cần
  • Khác với toFixed(): xác định tổng số chữ số có nghĩa
1.4 Hàm Number()

Mô tả: Ép kiểu giá trị về kiểu Number

Cú pháp: Number(value)

Ví dụ Number()
let str1 = "123.45";
let str2 = "123.45abc";
let str3 = "abc";
let str4 = "";
let bool = true;

console.log(Number(str1)); // 123.45
console.log(Number(str2)); // NaN (Not a Number)
console.log(Number(str3)); // NaN
console.log(Number(str4)); // 0
console.log(Number(bool)); // 1 (true → 1, false → 0)
console.log(Number(null)); // 0
console.log(Number(undefined)); // NaN
Phân tích:
  • Chuyển đổi bất kỳ giá trị nào sang kiểu Number
  • Trả về NaN nếu không thể chuyển đổi
  • Chuỗi trống và null chuyển thành 0
  • undefined chuyển thành NaN
  • Boolean: true → 1, false → 0
1.5 Hàm parseInt() và parseFloat()

Mô tả: Phân tích chuỗi thành số nguyên hoặc số thực

Cú pháp: parseInt(string, [radix]) | parseFloat(string)

Ví dụ parseInt() và parseFloat()
// parseInt - Chuyển chuỗi thành số nguyên
console.log(parseInt("123")); // 123
console.log(parseInt("123abc")); // 123
console.log(parseInt("abc123")); // NaN
console.log(parseInt("1010", 2)); // 10 (chuyển từ hệ 2)
console.log(parseInt("FF", 16)); // 255 (chuyển từ hệ 16)

// parseFloat - Chuyển chuỗi thành số thực
console.log(parseFloat("123.45")); // 123.45
console.log(parseFloat("123.45abc")); // 123.45
console.log(parseFloat("abc123.45")); // NaN
Phân tích:
  • parseInt(): Đọc chuỗi từ trái sang phải, dừng khi gặp ký tự không phải số
  • parseFloat(): Tương tự nhưng cho phép dấu thập phân và ký tự 'e' (khoa học)
  • Cả hai trả về NaN nếu ký tự đầu tiên không phải số
  • parseInt() có thể nhận tham số radix (2-36) để chỉ định hệ cơ số
1.6 Hàm isNaN() và isFinite()

Mô tả: Kiểm tra giá trị số

Cú pháp: isNaN(value) | isFinite(value)

Ví dụ isNaN() và isFinite()
// isNaN() - Kiểm tra xem có phải là NaN không
console.log(isNaN(123)); // false
console.log(isNaN("123")); // false (chuyển đổi được)
console.log(isNaN("abc")); // true
console.log(isNaN(NaN)); // true

// isFinite() - Kiểm tra xem có phải số hữu hạn không
console.log(isFinite(123)); // true
console.log(isFinite("123")); // true
console.log(isFinite(Infinity)); // false
console.log(isFinite("abc")); // false
Phân tích:
  • isNaN(): Kiểm tra xem giá trị có phải NaN (sau khi chuyển đổi sang số)
  • isFinite(): Kiểm tra xem giá trị có phải số hữu hạn (không phải Infinity/-Infinity/NaN)
  • Number.isNaN() và Number.isFinite() là phiên bản nghiêm ngặt hơn (không tự chuyển đổi kiểu)
  • isFinite(null) trả về true (vì null chuyển thành 0)

2. String - Kiểu chuỗi

Định nghĩa: Đại diện cho chuỗi ký tự.

Ví dụ cơ bản về String
let name = "John Doe"; // sử dụng dấu nháy kép
let greeting = 'Hello'; // sử dụng dấu nháy đơn
let templateStr = `My name is ${name}`; // template string (ES6)
let multiLine = `Line 1
Line 2
Line 3`; // chuỗi nhiều dòng
2.1 Phương thức charAt() và charCodeAt()

Mô tả: Truy cập ký tự trong chuỗi

Cú pháp: string.charAt(index) | string.charCodeAt(index)

Ví dụ charAt() và charCodeAt()
let str = "Hello";

// charAt() - Lấy ký tự tại vị trí
console.log(str.charAt(0)); // "H"
console.log(str.charAt(4)); // "o"
console.log(str.charAt(10)); // "" (chuỗi rỗng nếu vượt độ dài)

// charCodeAt() - Mã ASCII của ký tự
console.log(str.charCodeAt(0)); // 72 (mã 'H')
console.log(str.charCodeAt(1)); // 101 (mã 'e')

// Cách viết hiện đại (ES5+)
console.log(str[0]); // "H"
console.log(str[4]); // "o"
Phân tích:
  • charAt(): Trả về ký tự tại vị trí chỉ định
  • charCodeAt(): Trả về mã Unicode của ký tự tại vị trí chỉ định
  • Chỉ số bắt đầu từ 0 (0 là ký tự đầu tiên)
  • Nếu chỉ số vượt quá độ dài chuỗi, charAt() trả về chuỗi rỗng, charCodeAt() trả về NaN
  • ES5+ có thể dùng cú pháp string[index] tương tự charAt()
2.2 Phương thức indexOf() và lastIndexOf()

Mô tả: Tìm vị trí của chuỗi con

Cú pháp: string.indexOf(searchValue, [fromIndex]) | string.lastIndexOf(searchValue, [fromIndex])

Ví dụ indexOf() và lastIndexOf()
let str = "Hello World, welcome to the World";

// indexOf() - Tìm từ đầu chuỗi
console.log(str.indexOf("World")); // 6
console.log(str.indexOf("world")); // -1 (phân biệt hoa thường)
console.log(str.indexOf("o")); // 4
console.log(str.indexOf("o", 5)); // 7 (tìm từ vị trí 5)

// lastIndexOf() - Tìm từ cuối chuỗi
console.log(str.lastIndexOf("World")); // 28
console.log(str.lastIndexOf("o")); // 29
console.log(str.lastIndexOf("o", 25)); // 23

// Kiểm tra sự tồn tại
if (str.indexOf("World") !== -1) {
    console.log("Tìm thấy 'World'");
}
Phân tích:
  • indexOf(): Trả về vị trí đầu tiên tìm thấy chuỗi con
  • lastIndexOf(): Trả về vị trí cuối cùng tìm thấy chuỗi con
  • Trả về -1 nếu không tìm thấy
  • Tham số thứ hai (fromIndex) là vị trí bắt đầu tìm kiếm
  • Phân biệt chữ hoa chữ thường
  • Có thể dùng để kiểm tra sự tồn tại của chuỗi con
2.3 Phương thức includes(), startsWith(), endsWith()

Mô tả: Kiểm tra chuỗi con (ES6)

Cú pháp: string.includes(searchString, [position]) | string.startsWith(searchString, [position]) | string.endsWith(searchString, [length])

Ví dụ includes(), startsWith(), endsWith()
let str = "Hello World";

// includes() - Kiểm tra có chứa chuỗi con không
console.log(str.includes("World")); // true
console.log(str.includes("world")); // false (phân biệt hoa thường)
console.log(str.includes("o W")); // true
console.log(str.includes("Hello", 1)); // false (tìm từ vị trí 1)

// startsWith() - Kiểm tra bắt đầu bằng chuỗi con
console.log(str.startsWith("Hello")); // true
console.log(str.startsWith("World")); // false
console.log(str.startsWith("World", 6)); // true (từ vị trí 6)

// endsWith() - Kiểm tra kết thúc bằng chuỗi con
console.log(str.endsWith("World")); // true
console.log(str.endsWith("Hello")); // false
console.log(str.endsWith("Hello", 5)); // true (chỉ xét 5 ký tự đầu)
Phân tích:
  • includes(): Trả về true nếu chuỗi chứa chuỗi con
  • startsWith(): Trả về true nếu chuỗi bắt đầu bằng chuỗi con
  • endsWith(): Trả về true nếu chuỗi kết thúc bằng chuỗi con
  • Trả về boolean (true/false), không trả về vị trí
  • Tham số thứ hai cho phép chỉ định vị trí bắt đầu/độ dài kiểm tra
  • Phân biệt chữ hoa chữ thường
2.4 Phương thức slice(), substring(), substr()

Mô tả: Cắt chuỗi con từ chuỗi

Cú pháp: string.slice(start, [end]) | string.substring(start, [end]) | string.substr(start, [length])

Ví dụ slice(), substring(), substr()
let str = "Hello World";

// slice() - Cắt chuỗi theo vị trí (nhận số âm)
console.log(str.slice(0, 5)); // "Hello"
console.log(str.slice(6)); // "World" (từ vị trí 6 đến hết)
console.log(str.slice(-5)); // "World" (5 ký tự từ cuối)
console.log(str.slice(6, -1)); // "Worl" (từ 6 đến ký tự cuối trừ 1)

// substring() - Tương tự slice nhưng không nhận số âm
console.log(str.substring(0, 5)); // "Hello"
console.log(str.substring(6)); // "World"
console.log(str.substring(6, 11)); // "World"
console.log(str.substring(11, 6)); // "World" (tự động đổi chỗ)

// substr() - Cắt từ vị trí với độ dài (cũ, không nên dùng)
console.log(str.substr(6, 5)); // "World" (từ vị trí 6, lấy 5 ký tự)
console.log(str.substr(-5, 5)); // "World" (5 ký tự từ cuối)
Phân tích:
  • slice(): Linh hoạt nhất, hỗ trợ số âm (đếm từ cuối)
  • substring(): Không hỗ trợ số âm, tự động đổi chỗ nếu start > end
  • substr(): Dùng độ dài thay vì vị trí kết thúc (không nên dùng nữa)
  • Chỉ số bắt đầu từ 0
  • Kết quả không bao gồm ký tự tại vị trí end
  • Nên dùng slice() cho hầu hết trường hợp
2.5 Phương thức toUpperCase() và toLowerCase()

Mô tả: Chuyển đổi chữ hoa/chữ thường

Cú pháp: string.toUpperCase() | string.toLowerCase()

Ví dụ toUpperCase() và toLowerCase()
let str = "Hello World";

// toUpperCase() - Chuyển thành chữ hoa
console.log(str.toUpperCase()); // "HELLO WORLD"
console.log("héllo".toUpperCase()); // "HÉLLO" (hỗ trợ Unicode)

// toLowerCase() - Chuyển thành chữ thường
console.log(str.toLowerCase()); // "hello world"
console.log("HÉLLO".toLowerCase()); // "héllo"

// Ứng dụng: So sánh không phân biệt hoa thường
let input = "YES";
if (input.toLowerCase() === "yes") {
    console.log("Người dùng đồng ý");
}
Phân tích:
  • toUpperCase(): Chuyển tất cả ký tự thành chữ hoa
  • toLowerCase(): Chuyển tất cả ký tự thành chữ thường
  • Trả về chuỗi mới, không thay đổi chuỗi gốc
  • Hỗ trợ Unicode/đa ngôn ngữ
  • Thường dùng để chuẩn hóa dữ liệu nhập hoặc so sánh không phân biệt hoa thường
  • Không ảnh hưởng đến ký tự không phải chữ cái
2.6 Phương thức trim(), trimStart(), trimEnd()

Mô tả: Xóa khoảng trắng ở đầu/cuối chuỗi

Cú pháp: string.trim() | string.trimStart() | string.trimEnd()

Ví dụ trim(), trimStart(), trimEnd()
let spaced = " Hello World ";
let tabString = "\tHello\n";

// trim() - Xóa khoảng trắng cả đầu và cuối
console.log("'" + spaced.trim() + "'"); // 'Hello World'
console.log("'" + tabString.trim() + "'"); // 'Hello'

// trimStart() (hoặc trimLeft()) - Xóa khoảng trắng đầu
console.log("'" + spaced.trimStart() + "'"); // 'Hello World '

// trimEnd() (hoặc trimRight()) - Xóa khoảng trắng cuối
console.log("'" + spaced.trimEnd() + "'"); // ' Hello World'

// Ứng dụng: Xử lý input người dùng
let userInput = " john@example.com ";
let cleanedInput = userInput.trim();
console.log(`Email: ${cleanedInput}`); // "Email: john@example.com"
Phân tích:
  • trim(): Xóa khoảng trắng cả đầu và cuối chuỗi
  • trimStart()/trimLeft(): Xóa khoảng trắng ở đầu chuỗi
  • trimEnd()/trimRight(): Xóa khoảng trắng ở cuối chuỗi
  • Khoảng trắng bao gồm: space, tab, newline, và các ký tự whitespace Unicode
  • Trả về chuỗi mới, không thay đổi chuỗi gốc
  • Rất quan trọng khi xử lý input người dùng
  • trimLeft() và trimRight() là tên cũ, nên dùng trimStart() và trimEnd()
2.7 Phương thức padStart() và padEnd()

Mô tả: Thêm ký tự vào đầu/cuối chuỗi cho đủ độ dài (ES2017)

Cú pháp: string.padStart(targetLength, [padString]) | string.padEnd(targetLength, [padString])

Ví dụ padStart() và padEnd()
// padStart() - Thêm vào đầu chuỗi
console.log("5".padStart(3, "0")); // "005"
console.log("123".padStart(5, "0")); // "00123"
console.log("abc".padStart(6, "-")); // "---abc"
console.log("hello".padStart(3, "*")); // "hello" (độ dài đã đủ)

// padEnd() - Thêm vào cuối chuỗi
console.log("5".padEnd(3, "0")); // "500"
console.log("Hello".padEnd(10, "!")); // "Hello!!!!!"
console.log("abc".padEnd(5, "123")); // "abc12"

// Ứng dụng: Định dạng số và văn bản
let numbers = [1, 12, 123, 1234];
numbers.forEach(num => {
    console.log(num.toString().padStart(6, " "));
});
// Kết quả:
// " 1"
// " 12"
// " 123"
// " 1234"
Phân tích:
  • padStart(): Thêm ký tự vào đầu cho đến khi đạt độ dài target
  • padEnd(): Thêm ký tự vào cuối cho đến khi đạt độ dài target
  • targetLength: Độ dài chuỗi mong muốn sau khi pad
  • padString: Chuỗi dùng để pad (mặc định là space)
  • Nếu chuỗi đã dài hơn targetLength, trả về chuỗi gốc
  • padString sẽ được lặp lại nếu cần
  • Rất hữu ích để định dạng hiển thị
2.8 Phương thức split()

Mô tả: Chia chuỗi thành mảng các chuỗi con

Cú pháp: string.split([separator], [limit])

Ví dụ split()
let str = "apple,banana,orange,grape";

// Chia bằng dấu phẩy
console.log(str.split(","));
// Kết quả: ["apple", "banana", "orange", "grape"]

// Giới hạn số phần tử
console.log(str.split(",", 2));
// Kết quả: ["apple", "banana"]

// Chia bằng regex
let sentence = "Hello World! How are you?";
console.log(sentence.split(/\s+/));
// Kết quả: ["Hello", "World!", "How", "are", "you?"]

// Chia từng ký tự
console.log("Hello".split(""));
// Kết quả: ["H", "e", "l", "l", "o"]

// Ứng dụng: Tách tên từ full name
let fullName = "Nguyen Van A";
let nameParts = fullName.split(" ");
console.log(`Họ: ${nameParts[0]}`); // "Họ: Nguyen"
console.log(`Tên đệm: ${nameParts[1]}`); // "Tên đệm: Van"
console.log(`Tên: ${nameParts[2]}`); // "Tên: A"
Phân tích:
  • separator: Chuỗi hoặc regex dùng để chia (nếu bỏ trống, trả về mảng chứa toàn bộ chuỗi)
  • limit: Giới hạn số phần tử trong mảng kết quả
  • Trả về mảng các chuỗi con
  • Chuỗi separator không được bao gồm trong kết quả
  • Nếu separator là chuỗi rỗng, chia thành từng ký tự
  • Có thể kết hợp với join() để thay thế chuỗi
  • Rất hữu ích để phân tích CSV, log files, v.v.
2.9 Phương thức replace() và replaceAll()

Mô tả: Thay thế chuỗi con trong chuỗi

Cú pháp: string.replace(searchValue, replaceValue) | string.replaceAll(searchValue, replaceValue)

Ví dụ replace() và replaceAll()
let str = "Hello World! World is beautiful.";

// replace() - Thay thế lần đầu tiên
console.log(str.replace("World", "Universe"));
// Kết quả: "Hello Universe! World is beautiful."

// replace() với regex flag g (toàn cục)
console.log(str.replace(/World/g, "Universe"));
// Kết quả: "Hello Universe! Universe is beautiful."

// replaceAll() - Thay thế tất cả (ES2021)
console.log(str.replaceAll("World", "Universe"));
// Kết quả: "Hello Universe! Universe is beautiful."

// Sử dụng hàm callback
console.log("hello world".replace(/\w+/g, (match) => {
    return match.charAt(0).toUpperCase() + match.slice(1);
}));
// Kết quả: "Hello World"

// Ứng dụng: Định dạng số điện thoại
let phone = "0123456789";
let formattedPhone = phone.replace(/(\d{3})(\d{3})(\d{4})/, "$1-$2-$3");
console.log(formattedPhone); // "012-345-6789"
Phân tích:
  • replace(): Thay thế lần xuất hiện đầu tiên của searchValue
  • replaceAll(): Thay thế tất cả lần xuất hiện của searchValue
  • searchValue: Có thể là chuỗi hoặc regex
  • replaceValue: Có thể là chuỗi hoặc hàm callback
  • Trả về chuỗi mới, không thay đổi chuỗi gốc
  • Với regex, có thể sử dụng capture groups ($1, $2, ...)
  • Hàm callback nhận các tham số: match, p1, p2, ..., offset, string
  • replaceAll() yêu cầu searchValue là chuỗi hoặc regex với flag 'g'
2.10 Template Literals (ES6)

Mô tả: Chuỗi mẫu với khả năng nhúng biểu thức

Cú pháp: `string text ${expression} string text`

Ví dụ Template Literals
let name = "John";
let age = 30;
let price = 19.99;

// Nhúng biến
console.log(`My name is ${name} and I am ${age} years old`);
// Kết quả: "My name is John and I am 30 years old"

// Tính toán trong template
console.log(`Next year I will be ${age + 1} years old`);
// Kết quả: "Next year I will be 31 years old"

// Gọi hàm trong template
console.log(`Name in uppercase: ${name.toUpperCase()}`);
// Kết quả: "Name in uppercase: JOHN"

// Template string nhiều dòng
let multiLine = `
Customer Information:
--------------------
Name: ${name}
Age: ${age}
Price: $${price.toFixed(2)}
`;
console.log(multiLine);

// Tagged templates
function highlight(strings, ...values) {
    let result = "";
    strings.forEach((string, i) => {
        result += string;
        if (i < values.length) {
            result += `<strong>${values[i]}</strong>`;
        }
    });
    return result;
}

let message = highlight`Hello ${name}, you are ${age} years old`;
console.log(message);
// Kết quả: "Hello <strong>John</strong>, you are <strong>30</strong> years old"
Phân tích:
  • Được bao quanh bởi backticks (`) thay vì nháy đơn/nháy kép
  • Cho phép nhúng biểu thức JavaScript với cú pháp ${expression}
  • Hỗ trợ chuỗi nhiều dòng mà không cần ký tự đặc biệt
  • Có thể lồng nhau (nested templates)
  • Tagged templates cho phép xử lý template với hàm tùy chỉnh
  • Biểu thức bên trong ${} có thể là bất kỳ biểu thức JavaScript hợp lệ nào
  • Tự động chuyển đổi giá trị thành chuỗi
  • Hiệu quả hơn và dễ đọc hơn nối chuỗi thông thường

3. Boolean - Kiểu logic

Định nghĩa: Chỉ có hai giá trị: true hoặc false.

Ví dụ về Boolean
let isOnline = true;
let hasLicense = false;
let isGreater = (5 > 3); // true
let isEqual = (5 === "5"); // false

// Ép kiểu về Boolean
console.log(Boolean(1)); // true
console.log(Boolean(0)); // false
console.log(Boolean("Hello")); // true
console.log(Boolean("")); // false
console.log(Boolean(null)); // false
console.log(Boolean(undefined)); // false
console.log(Boolean(NaN)); // false
Phân tích:
  • Chỉ có hai giá trị: true (đúng) và false (sai)
  • Dùng trong các biểu thức điều kiện và vòng lặp
  • Các giá trị "falsy" (chuyển thành false): false, 0, -0, 0n, "", null, undefined, NaN
  • Các giá trị "truthy" (chuyển thành true): Mọi giá trị khác falsy
  • Toán tử logic: && (AND), || (OR), ! (NOT)
  • Hàm Boolean() dùng để ép kiểu bất kỳ giá trị nào về boolean
  • Boolean là immutable (không thể thay đổi giá trị true/false)

4. Undefined và Null

4.1 Undefined - Biến chưa được gán giá trị
let testVar; // khai báo nhưng chưa gán giá trị
console.log(testVar); // undefined
console.log(typeof testVar); // "undefined"

let obj = {};
console.log(obj.nonExistent); // undefined (thuộc tính không tồn tại)

function noReturn() {
    // không có return
}
console.log(noReturn()); // undefined
Phân tích:
  • Là giá trị mặc định của biến đã khai báo nhưng chưa được gán giá trị
  • Là giá trị trả về của hàm không có return statement
  • Là giá trị của thuộc tính không tồn tại trong object
  • typeof undefined trả về "undefined"
  • Không nên gán undefined một cách thủ công, dùng null thay thế
  • Là giá trị falsy (chuyển thành false trong ngữ cảnh boolean)
4.2 Null - Giá trị không tồn tại
let emptyValue = null; // gán giá trị null
console.log(emptyValue); // null
console.log(typeof emptyValue); // "object" (đây là bug của JavaScript)

// So sánh undefined và null
console.log(null == undefined); // true (chỉ so sánh giá trị)
console.log(null === undefined); // false (so sánh cả kiểu dữ liệu)
Phân tích:
  • Là giá trị đại diện cho "không có giá trị" hoặc "không tồn tại"
  • Phải được gán một cách rõ ràng, không phải giá trị mặc định
  • typeof null trả về "object" (là bug nổi tiếng của JavaScript)
  • Để kiểm tra null, dùng value === null
  • Là giá trị falsy (chuyển thành false trong ngữ cảnh boolean)
  • Thường dùng để chỉ định rõ ràng rằng một biến không có giá trị
  • Khác với undefined: null là giá trị được gán, undefined là trạng thái mặc định
