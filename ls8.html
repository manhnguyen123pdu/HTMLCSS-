I. HÀM (FUNCTION) LÀ GÌ?

Định nghĩa:

Hàm là một khối mã (code) dùng để thực hiện một công việc cụ thể, có thể được định nghĩa một lần và tái sử dụng nhiều lần ở bất cứ đâu trong chương trình.

Mục đích sử dụng:

  • Tránh lặp lại code (DRY: Don't Repeat Yourself)
  • Tăng tính tổ chức, dễ đọc, dễ quản lý
  • Có thể chia chương trình lớn thành các phần nhỏ (module hóa)
  • Cho phép tái sử dụng logic nhiều nơi

Cú pháp tổng quát:

Cú pháp
function tenHam(thamSo1, thamSo2, ...) {
    // Thân hàm: chứa các lệnh sẽ thực thi
    return giaTriTraVe;
}

Giải thích cú pháp từng phần:

Thành phần Giải thích
function Từ khóa để khai báo một hàm
tenHam Tên định danh (tùy ý) để gọi lại hàm
(thamSo1, thamSo2, ...) Danh sách các tham số truyền vào (parameters)
{...} Thân hàm – nơi chứa các lệnh được thực hiện khi gọi hàm
return Từ khóa để trả giá trị ra khỏi hàm

Ví dụ minh hoạ:

Ví dụ hàm tính tổng
function tinhTong(a, b) {
    // a và b là các tham số nhận giá trị khi gọi hàm
    let tong = a + b; // Thực hiện tính tổng
    return tong; // Trả lại kết quả tổng
}

let kq = tinhTong(5, 10); // Gọi hàm với a=5, b=10 → nhận kết quả là 15
console.log(kq); // In ra: 15

Kết quả: 15

II. GỌI HÀM (FUNCTION CALL)

Định nghĩa:

Gọi hàm là hành động kích hoạt hàm để nó thực thi phần thân bên trong và (nếu có) trả lại kết quả.

Mục đích sử dụng:

  • Thực hiện logic đã viết trong hàm
  • Có thể gọi bao nhiêu lần cũng được
  • Truyền đối số khác nhau → nhận kết quả khác nhau

Cú pháp:

Cú pháp
tenHam(doiSo1, doiSo2, ...);

Giải thích:

Thành phần Ý nghĩa
tenHam Tên hàm đã khai báo trước đó
(doiSo1, doiSo2, ...) Đối số truyền vào (arguments) để xử lý

Ví dụ có giải thích:

Ví dụ
function chao(ten) {
    console.log("Chào bạn", ten);
}

chao("Lan"); // Gọi hàm và truyền "Lan" vào → In: Chào bạn Lan
chao("Minh"); // In: Chào bạn Minh

III. return – GIÁ TRỊ TRẢ VỀ

Định nghĩa:

return là từ khóa dùng để trả giá trị từ hàm ra ngoài. Khi gặp return, hàm sẽ kết thúc ngay lập tức.

Mục đích:

  • Trả kết quả xử lý về cho nơi gọi hàm
  • Có thể lưu trữ kết quả để tiếp tục xử lý

Cú pháp:

Cú pháp
return bieuThucTraVe;

Giải thích:

Thành phần Ý nghĩa
return Kết thúc hàm và trả lại kết quả
bieuThucTraVe Giá trị cụ thể (số, chuỗi, biến, phép tính, ...)

Ví dụ:

Ví dụ
function binhPhuong(x) {
    return x * x; // Trả về bình phương
}
let kq = binhPhuong(7); // Nhận kết quả 49
console.log(kq); // In ra: 49

IV. HÀM KHÔNG TRẢ GIÁ TRỊ (VOID FUNCTION)

Định nghĩa:

Là hàm không có từ khóa return, chỉ thực hiện một hành động nào đó (in ra, thay đổi dữ liệu, …)

Mục đích:

  • In ra màn hình
  • Cập nhật dữ liệu
  • Gửi thông báo,...

Cú pháp:

Cú pháp
function tenHam() {
    // Thực hiện công việc nhưng không return
}

Giải thích:

Nếu không có return, hàm sẽ trả về undefined mặc định

Ví dụ:

Ví dụ
function thongBao() {
    console.log("Hệ thống đang khởi động...");
}

let x = thongBao(); // In ra nhưng x là undefined
console.log(x); // undefined

V. Arrow Function (Hàm mũi tên)

Định nghĩa:

Arrow function là cách viết ngắn gọn của hàm trong JavaScript (ES6+), thường dùng trong các tình huống đơn giản, đặc biệt khi xử lý callback hoặc function ngắn.

Mục đích:

  • Viết hàm gọn hơn
  • Tự động bind this theo ngữ cảnh
  • Thuận tiện khi xử lý các logic nhỏ, dùng làm callback

Cú pháp:

Cú pháp
const tenHam = (thamSo1, thamSo2, ...) => {
    // Thân hàm
    return giaTri;
};

// Nếu chỉ có 1 dòng return, có thể viết ngắn:
const tenHam = (a, b) => a + b;

Giải thích:

Thành phần Ý nghĩa
const tenHam Khai báo hàm với từ khóa const
(a, b) Danh sách tham số
=> Biểu thị là hàm mũi tên
{...} Thân hàm (nếu nhiều dòng)
return Có thể viết tường minh hoặc ẩn nếu một dòng

Ví dụ:

Ví dụ
const binhPhuong = x => x * x;
console.log(binhPhuong(6)); // In ra 36

const chao = (ten) => {
    console.log(`Xin chào ${ten}!`);
};
chao("Lan"); // In: Xin chào Lan!

VI. Function Expression (Hàm biểu thức)

Định nghĩa:

Function expression là hàm được gán cho một biến (tức là khai báo gián tiếp), không cần tên hàm.

Mục đích:

  • Dùng như biến
  • Truyền vào đối số, hoặc callback
  • Có thể định nghĩa trong phạm vi nhỏ hơn (scope)

Cú pháp:

Cú pháp
const tenBien = function(thamSo) {
    // Thân hàm
    return giaTri;
};

Giải thích:

Thành phần Ý nghĩa
function Từ khóa
tenBien Tên biến chứa hàm
(thamSo) Tham số truyền vào
{} Thân hàm

Ví dụ:

Ví dụ
const tinhHieu = function(a, b) {
    return a - b;
};

console.log(tinhHieu(10, 3)); // In: 7

VII. Tham số mặc định (Default Parameters)

Định nghĩa:

Cho phép gán giá trị mặc định cho tham số nếu không truyền đối số khi gọi hàm.

Mục đích:

  • Tránh lỗi khi thiếu đối số
  • Viết code linh hoạt hơn

Cú pháp:

Cú pháp
function tenHam(thamSo = giaTriMacDinh) {
    //...
}

Giải thích:

Nếu đối số bị bỏ qua hoặc undefined, tham số sẽ nhận giá trị mặc định.

Ví dụ:

Ví dụ
function sayHello(ten = "bạn") {
    console.log("Xin chào", ten);
}

sayHello("Minh"); // In: Xin chào Minh
sayHello(); // In: Xin chào bạn

VIII. Đệ quy (Recursion)

Định nghĩa:

Là hàm gọi lại chính nó bên trong để giải quyết một bài toán lặp lại có cấu trúc lồng nhau.

Mục đích:

Thay cho vòng lặp khi xử lý bài toán phân tách nhỏ (ví dụ: giai thừa, dãy Fibonacci, cấu trúc cây)

Cú pháp:

Cú pháp
function tenHam(dieuKienDung) {
    if (dieuKienDung) return;
    // Gọi lại chính nó
    tenHam(giaTriMoi);
}

Giải thích:

Luôn cần điều kiện dừng để tránh lặp vô hạn

Ví dụ:

Ví dụ
function giaiThua(n) {
    if (n === 0) return 1; // Điều kiện dừng
    return n * giaiThua(n - 1); // Gọi đệ quy
}

console.log(giaiThua(5)); // In: 120

IX. Callback Function

Định nghĩa:

Callback là hàm được truyền như một đối số vào một hàm khác, và được gọi lại (callback) bên trong hàm đó.

Mục đích:

  • Cho phép xử lý linh hoạt (đặc biệt trong bất đồng bộ)
  • Thường dùng trong forEach, map, filter, xử lý AJAX, API...

Cú pháp:

Cú pháp
function xuLy(callback) {
    callback(); // Gọi lại hàm truyền vào
}

Giải thích:

callback chính là một hàm được truyền như đối số
Có thể gọi lại bất cứ lúc nào

Ví dụ:

Ví dụ
function xinChao(callback) {
    console.log("Xin chào!");
    callback();
}

function loiChaoPhu() {
    console.log("Chúc bạn một ngày vui vẻ!");
}

xinChao(loiChaoPhu);
// In:
// Xin chào!
// Chúc bạn một ngày vui vẻ!

X. Tham trị (Pass by Value) & Tham chiếu (Pass by Reference)

Tham trị (Pass by Value)

Định nghĩa:

Với các kiểu dữ liệu nguyên thuỷ (number, string, boolean, undefined, null, symbol, bigint), khi truyền vào hàm, giá trị được sao chép ra bản sao riêng. Việc thay đổi giá trị trong hàm không ảnh hưởng tới biến ban đầu.

Cú pháp & ví dụ:

Ví dụ tham trị
function thayDoi(a) {
    a = 100;
}

let x = 5;
thayDoi(x);
console.log(x); // 5

Giải thích:

x = 5 là giá trị nguyên thủy (number)
thayDoi(x) truyền giá trị 5 vào tham số a, a là bản sao nên đổi a = 100 không ảnh hưởng x

Tham chiếu (Pass by Reference)

Định nghĩa:

Với object, array, function, giá trị được truyền vào là một tham chiếu tới vùng nhớ. Khi thay đổi nội dung bên trong, thì giá trị gốc cũng bị ảnh hưởng.

Cú pháp & ví dụ:

Ví dụ tham chiếu
function capNhat(obj) {
    obj.ten = "Lan";
}

let user = { ten: "Mai" };
capNhat(user);
console.log(user.ten); // "Lan"

Giải thích:

user là một object
obj trong hàm capNhat trỏ cùng vùng nhớ với user, nên khi gán obj.ten = "Lan" thì user.ten cũng đổi theo.

XI. Rest Parameter (...args)

Định nghĩa:

Giúp gom nhiều đối số truyền vào hàm thành một mảng.

Mục đích:

  • Viết hàm nhận số lượng tham số không cố định.
  • Linh hoạt xử lý dữ liệu động.

Cú pháp & ví dụ:

Ví dụ
function tinhTong(...args) {
    return args.reduce((tong, so) => tong + so, 0);
}

console.log(tinhTong(1, 2, 3, 4)); // 10

Giải thích cú pháp:

...args gom các đối số thành mảng [1, 2, 3, 4]
Dùng .reduce() để cộng dồn

XII. Spread Operator (...)

Định nghĩa:

Dùng để trải mảng (hoặc object) thành các phần tử riêng lẻ.

Mục đích:

  • Truyền phần tử mảng vào hàm
  • Sao chép, gộp mảng/object dễ dàng

Ví dụ:

Ví dụ
function cong(a, b, c) {
    return a + b + c;
}

let nums = [1, 2, 3];
console.log(cong(...nums)); // 6

Giải thích:

...nums tương đương cong(1, 2, 3)

XIII. IIFE – Immediately Invoked Function Expression

Định nghĩa:

Hàm được định nghĩa và gọi ngay lập tức.

Mục đích:

  • Tạo phạm vi riêng (scope riêng), tránh xung đột biến
  • Thường dùng trong module hoặc script tự chạy

Cú pháp & ví dụ:

Ví dụ
(function () {
    console.log("Chạy ngay lập tức");
})();

Giải thích:

() đầu tiên bao hàm định nghĩa
() sau gọi hàm đó ngay

XIV. Hoisting trong hàm

Định nghĩa:

Hoisting là cơ chế đưa khai báo hàm/biến lên đầu phạm vi (scope) trước khi code được thực thi.

So sánh các loại hàm:

Loại Có hoisting? Gọi trước khi khai báo?
function declaration Có Được
function expression Không Lỗi
arrow function Không Lỗi

Ví dụ:

Ví dụ
sayHello(); // OK

function sayHello() {
    console.log("Hello");
}

// sayHi(); Lỗi
const sayHi = function () {
    console.log("Hi");
};

Giải thích:

sayHello được hoisting nên gọi trước được
sayHi là gán hàm cho biến, biến chỉ được hoisting phần khai báo (undefined), nên lỗi khi gọi trước

XV. Hàm trong hàm (Nested Function)

Định nghĩa:

Một hàm có thể được khai báo bên trong một hàm khác.

Mục đích:

  • Tổ chức code rõ ràng
  • Tạo phạm vi riêng cho hàm con

Cú pháp & ví dụ:

Ví dụ
function outer() {
    function inner() {
        console.log("Bên trong");
    }
    inner();
}

outer(); // "Bên trong"

Giải thích:

inner() chỉ tồn tại trong outer(), không gọi được từ ngoài

XVI. Closure – Ghi nhớ phạm vi cũ

Định nghĩa:

Closure là hàm con ghi nhớ biến cục bộ của hàm cha dù hàm cha đã kết thúc.

Mục đích:

  • Lưu trạng thái
  • Bảo vệ biến nội bộ
  • Dùng trong tạo hàm động

Ví dụ:

Ví dụ
function taoBoDem() {
    let dem = 0;
    return function() {
        dem++;
        console.log(dem);
    };
}

const dem1 = taoBoDem();
dem1(); // 1
dem1(); // 2

Giải thích:

dem không bị mất sau khi taoBoDem() kết thúc
Closure giữ lại môi trường dem => đây là cơ chế Closure

Bài 3: Quản lý sinh viên

Viết chương trình JS để quản lý danh sách sinh viên. Mỗi sinh viên gồm các thông tin:

  • id – mã sinh viên (chuỗi)
  • name – họ tên
  • gender – giới tính
  • score – điểm số (0–10)
Dữ liệu mẫu
let students = [
  { id: "SV001", name: "Nguyễn Văn A", gender: "Nam", score: 8.5 },
  { id: "SV002", name: "Trần Thị B", gender: "Nữ", score: 6.7 },
  { id: "SV003", name: "Lê Văn C", gender: "Nam", score: 4.8 }
];

Chức năng: (yêu cầu viết hàm)

  • Hiển thị danh sách sinh viên (gồm loại học lực: Giỏi, Khá, TB, Yếu)
  • Thêm sinh viên mới từ bàn phím
  • Tìm sinh viên theo tên (gợi ý: includes)
  • Xóa sinh viên theo mã
  • Menu chức năng sử dụng vòng lặp

Áp dụng hàm, mảng, object, câu điều kiện, toán tử, vòng lặp
