I. Object là gì?

Định nghĩa:

Object là một kiểu dữ liệu trong JavaScript cho phép lưu trữ nhiều giá trị dưới dạng cặp key-value (thuộc tính: giá trị). Mỗi key (còn gọi là "property name") thường là chuỗi, còn value có thể là bất kỳ kiểu dữ liệu nào, kể cả function.

Mục đích:

  • Gom nhóm các thông tin liên quan về một đối tượng thực tế (người, sản phẩm, xe, v.v).
  • Dễ quản lý và truy cập dữ liệu có cấu trúc phức tạp.
  • Là nền tảng cho lập trình hướng đối tượng trong JavaScript.

Cú pháp:

Cú pháp cơ bản
const objectName = {
    key1: value1,
    key2: value2,
    ...
};

Giải thích cú pháp:

  • objectName: Tên của đối tượng (do bạn đặt).
  • key1, key2: Tên thuộc tính (phải là chuỗi, nếu không thì được ép thành chuỗi).
  • value1, value2: Giá trị tương ứng với từng key. Có thể là chuỗi, số, boolean, array, object khác hoặc function.

Ví dụ:

Ví dụ object đơn giản
const person = {
    name: "Nam",
    age: 25,
    isStudent: true
};

II. Cách tạo Object

1. Dùng cú pháp literal (phổ biến nhất)

Định nghĩa: Tạo object bằng dấu {} và liệt kê các cặp key: value bên trong.

Mục đích: Nhanh chóng, ngắn gọn khi khai báo một object tĩnh.

Cú pháp:

Object literal
const objectName = {
    key1: value1,
    key2: value2
};

Giải thích cú pháp:

  • Cặp key-value được khai báo trong dấu {}.
  • Các cặp cách nhau bằng dấu phẩy ,.

Ví dụ:

Ví dụ object literal
const book = {
    title: "JavaScript Cơ bản",
    pages: 120
};

2. Dùng từ khóa new Object()

Định nghĩa: Tạo một object mới bằng cách gọi constructor Object().

Mục đích: Tạo object khi bạn muốn thêm thuộc tính sau từng bước.

Cú pháp:

new Object()
const obj = new Object();
obj.key = value;

Giải thích cú pháp:

  • new Object() tạo ra một object rỗng.
  • Các thuộc tính được thêm vào như obj.key = value.

Ví dụ:

Ví dụ new Object()
const student = new Object();
student.name = "Linh";
student.age = 21;

3. Dùng Constructor Function (Hàm khởi tạo)

Định nghĩa: Sử dụng một hàm đặc biệt để tạo ra các object có cùng cấu trúc.

Mục đích: Tạo nhiều object giống nhau về cấu trúc.

Cú pháp:

Constructor Function
function ConstructorName(param1, param2) {
    this.key1 = param1;
    this.key2 = param2;
}
const obj = new ConstructorName(arg1, arg2);

Giải thích cú pháp:

  • this.key = param: gán giá trị vào object mới.
  • new giúp tạo ra instance mới.
  • Instance là một bản sao cụ thể được tạo ra từ một "khuôn mẫu" (constructor function hoặc class).

Ví dụ:

Ví dụ Constructor Function
function Car(brand, color) {
    this.brand = brand;
    this.color = color;
}
const myCar = new Car("Toyota", "Red");

4. Dùng Class (ES6+)

Định nghĩa: Class là cách viết hiện đại để tạo đối tượng với phương thức và thuộc tính.

Mục đích: Tổ chức code rõ ràng, dễ mở rộng.

Cú pháp:

Class ES6
class ClassName {
    constructor(param1, param2) {
        this.key1 = param1;
        this.key2 = param2;
    }
}
const obj = new ClassName(arg1, arg2);

Giải thích cú pháp:

  • constructor: hàm chạy khi dùng new.
  • this: đại diện cho object được tạo.

Ví dụ:

Ví dụ Class
class Student {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}
const s1 = new Student("Quang", 20);

III. Thuộc tính và phương thức trong Object

1. Thuộc tính (Property)

Định nghĩa: Thuộc tính là cặp key: value được lưu trong object. Mỗi thuộc tính là một phần thông tin mô tả cho đối tượng.

Mục đích: Lưu trữ dữ liệu liên quan đến đối tượng cụ thể. Dễ dàng truy cập, sửa đổi hoặc hiển thị thông tin.

Cú pháp:

Truy cập thuộc tính
objectName.key // Truy cập bằng dấu chấm (dot notation)
objectName["key"] // Truy cập bằng dấu ngoặc vuông (bracket notation)

Giải thích cú pháp:

  • objectName.key: dùng khi key là một tên hợp lệ (không có khoảng trắng, dấu đặc biệt).
  • objectName["key"]: dùng khi key là biến, chuỗi hoặc có ký tự đặc biệt.

Ví dụ:

Truy cập thuộc tính
const user = {
    name: "Hưng",
    age: 28
};

console.log(user.name); // "Hưng"
console.log(user["age"]); // 28

const key = "name";
console.log(user[key]); // "Hưng"

2. Thêm hoặc sửa thuộc tính

Định nghĩa: Có thể thêm thuộc tính mới hoặc cập nhật giá trị thuộc tính đã có.

Mục đích: Linh hoạt mở rộng dữ liệu trong object bất kỳ lúc nào.

Cú pháp:

Thêm/sửa thuộc tính
objectName.newKey = value;
objectName["newKey"] = value;

Giải thích cú pháp:

  • Nếu newKey chưa tồn tại, nó sẽ được thêm mới.
  • Nếu newKey đã tồn tại, giá trị sẽ bị ghi đè.

Ví dụ:

Thêm/sửa thuộc tính
const product = {
    name: "Laptop"
};

product.price = 1500; // thêm mới
product.name = "Gaming Laptop"; // sửa

console.log(product); // { name: "Gaming Laptop", price: 1500 }

3. Xóa thuộc tính

Định nghĩa: Dùng từ khóa delete để loại bỏ một thuộc tính khỏi object.

Mục đích: Dọn dẹp hoặc ẩn dữ liệu không còn cần thiết.

Cú pháp:

Xóa thuộc tính
delete objectName.key;
delete objectName["key"];

Giải thích cú pháp:

  • Sau khi xóa, thuộc tính đó không còn tồn tại trong object nữa.

Ví dụ:

Xóa thuộc tính
const user = {
    name: "Huy",
    age: 30
};

delete user.age;
console.log(user); // { name: "Huy" }

4. Phương thức trong object (Method)

Định nghĩa: Phương thức là một thuộc tính có giá trị là hàm. Dùng để thực hiện hành vi (hành động) của object.

Mục đích: Gắn hành vi vào object, giúp quản lý hành vi và dữ liệu cùng nhau. Là nền tảng cho Lập trình hướng đối tượng (OOP).

Cú pháp:

Phương thức
const object = {
    methodName: function() {
        // code
    }
};

//Hoặc cú pháp rút gọn (ES6):

const object = {
    methodName() {
        // code
    }
};

Giải thích cú pháp:

  • methodName: tên của phương thức.
  • function() {...}: hàm định nghĩa hành vi.
  • Có thể sử dụng this để tham chiếu đến chính object đó.

Ví dụ:

Ví dụ phương thức
const dog = {
    name: "Lu",
    speak: function() {
        console.log(`${this.name} nói: Gâu Gâu!`);
    }
};

dog.speak(); // "Lu nói: Gâu Gâu!"

// Hoặc viết ngắn hơn:
const cat = {
    name: "Miu",
    speak() {
        console.log(`${this.name} kêu: Meo Meo!`);
    }
};
cat.speak(); // "Miu kêu: Meo Meo!"

IV. Duyệt Object (Lặp qua thuộc tính)

Định nghĩa:

Duyệt object là quá trình truy cập lần lượt từng cặp key: value trong một đối tượng.

Mục đích:

  • Hiển thị toàn bộ thông tin trong object.
  • Thực hiện thao tác với từng thuộc tính như kiểm tra, tính toán, lọc...

Cú pháp:

Cách 1: for...in (truyền thống)
for...in
for (let key in object) {
    console.log(key, object[key]);
}
Cách 2: Dùng Object.keys, Object.values, Object.entries
Object methods
Object.keys(obj); // → mảng các key
Object.values(obj); // → mảng các value
Object.entries(obj); // → mảng các cặp [key, value]

Giải thích cú pháp:

  • for...in: duyệt qua từng thuộc tính (bao gồm kế thừa).
  • Object.keys() trả về mảng chứa tất cả key của object.
  • Object.entries() dùng với for...of để vừa lấy key vừa lấy value.

Ví dụ:

Ví dụ duyệt object
const student = {
    name: "Hải",
    age: 23,
    score: 8.5
};

// Dùng for...in
for (let key in student) {
    console.log(`${key}: ${student[key]}`);
}

// Dùng Object.entries
for (let [key, value] of Object.entries(student)) {
    console.log(`${key} => ${value}`);
}

V. Destructuring Object (Phân rã đối tượng)

Định nghĩa:

Destructuring là cú pháp giúp trích xuất dữ liệu từ object ra biến một cách nhanh chóng.

Mục đích:

  • Truy xuất nhiều thuộc tính một cách ngắn gọn.
  • Code ngắn gọn, dễ đọc hơn khi làm việc với object có nhiều thuộc tính.

Cú pháp:

Destructuring
const { key1, key2 } = object;

Giải thích cú pháp:

  • key1, key2: tên thuộc tính cần trích xuất.
  • Phải trùng tên với key trong object.
  • Có thể đổi tên: { key1: newVarName }.

Ví dụ:

Ví dụ destructuring
const product = {
    name: "Mouse",
    price: 200,
    brand: "Logitech"
};

const { name, price } = product;
console.log(name); // "Mouse"
console.log(price); // 200

// Đổi tên biến
const { brand: productBrand } = product;
console.log(productBrand); // "Logitech"

VI. Copy Object (Sao chép đối tượng)

Định nghĩa:

Copy object là tạo ra một bản sao từ object gốc.

Mục đích:

  • Tránh ảnh hưởng đến object gốc khi thay đổi bản sao.
  • Làm việc với dữ liệu độc lập.

Cú pháp:

1. Copy bằng tham chiếu (KHÔNG độc lập)
Copy tham chiếu
const newObj = oldObj;
2. Shallow copy (sao chép nông)
Shallow copy
const copy1 = Object.assign({}, obj);
const copy2 = { ...obj };
3. Deep copy (sao chép sâu)
Deep copy
const deepCopy = JSON.parse(JSON.stringify(obj));

Giải thích cú pháp:

  • Copy tham chiếu: cả hai biến cùng trỏ đến một object trong bộ nhớ.
  • Shallow copy: chỉ sao chép các thuộc tính cấp 1, object lồng bên trong vẫn bị tham chiếu.
  • Deep copy: tạo bản sao hoàn toàn tách biệt kể cả object lồng nhau.

Ví dụ:

Ví dụ copy object
const person = {
    name: "An",
    address: {
        city: "Hanoi"
    }
};

// Shallow copy
const copy = { ...person };
copy.name = "Bình";
copy.address.city = "Saigon";

console.log(person.name); // "An"
console.log(person.address.city); // "Saigon" ❗️bị ảnh hưởng

// Deep copy
const deepCopy = JSON.parse(JSON.stringify(person));
deepCopy.address.city = "Hue";
console.log(person.address.city); // "Saigon"
console.log(deepCopy.address.city); // "Hue"

VII. Optional Chaining (?.) & Nullish Coalescing (??)

Định nghĩa:

  • Optional chaining ?.: truy cập sâu trong object mà không gây lỗi nếu giá trị trung gian là undefined hoặc null.
  • Nullish coalescing ??: trả về giá trị bên phải nếu bên trái là null hoặc undefined.

Mục đích:

  • Tránh lỗi khi truy cập thuộc tính không tồn tại.
  • Đảm bảo gán giá trị mặc định an toàn.

Cú pháp:

Optional Chaining & Nullish Coalescing
object?.key?.subKey
let value = input ?? "default value";

Giải thích cú pháp:

  • obj?.prop: nếu obj là null hoặc undefined, sẽ trả về undefined mà không lỗi.
  • a ?? b: nếu a là null hoặc undefined, trả về b. Các giá trị như 0, false, "" vẫn giữ nguyên.

Ví dụ:

Ví dụ Optional Chaining và Nullish Coalescing
const user = {
    profile: {
        name: "Mai"
    }
};

console.log(user.profile?.name); // "Mai"
console.log(user.contact?.email); // undefined, không lỗi

const username = null;
const display = username ?? "Guest";
console.log(display); // "Guest"

VIII. Tạo Object động (Tính năng nâng cao)

1. Computed Property Names

Định nghĩa: Tạo key từ giá trị biến động (biến, kết quả tính toán).

Cú pháp:

Computed Property Names
const key = "email";
const obj = {
    [key]: "abc@gmail.com"
};

Ví dụ:

Ví dụ Computed Property Names
const prop = "age";
const user = {
    name: "Hoàng",
    [prop]: 27
};
console.log(user.age); // 27

2. Shorthand Property & Method (Viết gọn)

Định nghĩa: Viết tắt khi tên biến trùng với tên key, hoặc định nghĩa phương thức.

Cú pháp:

Shorthand Property & Method
const name = "Lan";
const person = { name }; // tương đương: { name: name }

const obj = {
    greet() {
        console.log("Hi");
    }
};

Ví dụ:

Ví dụ Shorthand
const price = 1000;
const product = {
    name: "TV",
    price, // tương đương price: price
    showInfo() {
        console.log(`${this.name} - ${this.price}`);
    }
};
product.showInfo(); // "TV - 1000"
