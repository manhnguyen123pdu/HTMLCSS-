PHẦN 1: GIỚI THIỆU VỀ ĐỒNG BỘ VÀ BẤT ĐỒNG BỘ

1.1. JavaScript là đơn luồng (Single-threaded)

Định nghĩa : JavaScript là ngôn ngữ đơn luồng, nghĩa là nó chỉ có một Call Stack duy nhất để thực thi code. Điều này có nghĩa là tại một thời điểm, JavaScript chỉ có thể làm một việc duy nhất.

Tưởng tượng:

  • Giống như một nhân viên ngân hàng chỉ phục vụ một khách hàng tại một thời điểm.
  • Hoặc như một ống hút, chỉ có thể uống một ngụm nước mỗi lần.

Minh họa bằng code:

javascript
function task1() {
    console.log("Bắt đầu công việc 1");
    // Giả sử công việc 1 mất 3 giây
    let start = Date.now();
    while (Date.now() - start < 3000) {} // Vòng lặp chặn
    console.log("Kết thúc công việc 1");
}

function task2() {
    console.log("Thực hiện công việc 2");
}

task1();
task2();

// Output:
// "Bắt đầu công việc 1"
// (3 giây sau)
// "Kết thúc công việc 1"
// "Thực hiện công việc 2"
// => task2 phải chờ task1 hoàn thành dù không liên quan

Ưu điểm của đơn luồng:

Đơn giản, dễ hiểu, không gặp vấn đề đồng bộ dữ liệu phức tạp như đa luồng (race condition, deadlock).

Nhược điểm:

Nếu có tác vụ nặng, toàn bộ chương trình bị "đóng băng".

1.2. Bài toán của xử lý đồng bộ (Blocking)

Định nghĩa Blocking: Blocking xảy ra khi một tác vụ tốn thời gian chặn không cho các tác vụ khác thực thi cho đến khi nó hoàn thành.

Ví dụ thực tế trong lập trình web:

html + javascript
<button id="myButton">Click me!</button>
<div id="status"></div>

<script>
    document.getElementById('myButton').addEventListener('click', function() {
        // Mô phỏng tác vụ nặng (lấy dữ liệu lớn, xử lý ảnh, v.v.)
        document.getElementById('status').innerText = 'Đang xử lý...';
        
        // Vòng lặp chặn trong 5 giây
        let start = Date.now();
        while (Date.now() - start < 5000) {
            // Không làm gì, chỉ chờ
        }
        
        document.getElementById('status').innerText = 'Hoàn thành!';
    });
    
    console.log('Trình duyệt đã sẵn sàng');
</script>

Hậu quả khi user click:

  • Nút bị "đơ", không thể click lại
  • Không thể scroll trang
  • Các animation dừng lại
  • Trải nghiệm người dùng cực kỳ tệ

Các tác vụ blocking phổ biến:

  • Đọc/ghi file lớn
  • Gửi request đến server
  • Xử lý dữ liệu phức tạp (hàng triệu bản ghi)
  • Mã hóa/giải mã dữ liệu

1.3. Khái niệm bất đồng bộ (Non-blocking)

Định nghĩa: Bất đồng bộ là kỹ thuật cho phép chương trình bắt đầu một tác vụ tốn thời gian, không chờ nó hoàn thành mà tiếp tục chạy các tác vụ khác. Khi tác vụ kia hoàn thành, chúng ta sẽ được thông báo và xử lý kết quả.

Ví dụ trực quan:

javascript
// Mô phỏng gọi món ở nhà hàng
console.log("Bước 1: Ngồi vào bàn");
console.log("Bước 2: Gọi món (đồ ăn mất 10 phút)");

// setTimeout mô phỏng việc nấu ăn
setTimeout(() => {
    console.log("Bước 4: Đồ ăn được mang ra");
}, 10000); // 10 giây

console.log("Bước 3: Nói chuyện với bạn bè trong lúc chờ");
// Output:
// Bước 1, Bước 2, Bước 3 (ngay lập tức)
// Bước 4 (sau 10 giây)

Lợi ích:

  • Ứng dụng luôn phản hồi nhanh
  • Tận dụng thời gian chờ để làm việc khác
  • Cải thiện trải nghiệm người dùng

PHẦN 2: CƠ CHẾ HOẠT ĐỘNG BẤT ĐỒNG BỘ (EVENT LOOP)

2.1. Call Stack

Định nghĩa chi tiết: Call Stack là một cấu trúc dữ liệu dạng ngăn xếp (LIFO - Last In First Out), dùng để theo dõi vị trí thực thi của chương trình.

Cách hoạt động:

  • Khi một hàm được gọi, nó được đẩy (push) vào đỉnh stack
  • Khi hàm kết thúc (return), nó được lấy ra (pop) khỏi stack
  • JavaScript chỉ thực thi hàm đang ở đỉnh stack

Minh họa chi tiết bằng diagram:

javascript
function multiply(a, b) {
    return a * b;
}

function square(n) {
    return multiply(n, n);
}

function printSquare(n) {
    const result = square(n);
    console.log(result);
}

printSquare(5);

Quá trình Call Stack thay đổi:

text
Bước 1: [printSquare(5)] // Gọi printSquare
Bước 2: [printSquare(5), square(5)] // Trong printSquare, gọi square
Bước 3: [printSquare(5), square(5), multiply(5,5)] // Trong square, gọi multiply
Bước 4: [printSquare(5), square(5)] // multiply kết thúc, pop
Bước 5: [printSquare(5)] // square kết thúc, pop
Bước 6: [] // printSquare kết thúc, pop

Lỗi Stack Overflow:

javascript
function infiniteRecursion() {
    infiniteRecursion(); // Gọi chính nó vô hạn
}

infiniteRecursion();
// Lỗi: RangeError: Maximum call stack size exceeded

2.2. Web APIs & Callback Queue (Task Queue)

Web APIs là gì?

Web APIs là các API do trình duyệt cung cấp, không phải của JavaScript engine. Bao gồm:

  • setTimeout, setInterval
  • fetch (gọi HTTP)
  • DOM events (click, scroll, keypress)
  • localStorage, sessionStorage
  • Geolocation API, Canvas API, v.v.

Cách hoạt động:

javascript
console.log('Bắt đầu');

// setTimeout được chuyển cho Web APIs xử lý
setTimeout(() => {
    console.log('Timeout hoàn thành');
}, 2000);

console.log('Kết thúc');

// Phân tích:
// 1. Call Stack: chạy console.log('Bắt đầu')
// 2. Gặp setTimeout, chuyển cho Web APIs (hẹn giờ 2s)
// 3. Call Stack: chạy console.log('Kết thúc')
// 4. Web APIs đếm ngược 2s
// 5. Sau 2s, callback được đưa vào Callback Queue
// 6. Khi Call Stack rỗng, Event Loop đưa callback vào Call Stack
// 7. Thực thi callback, in 'Timeout hoàn thành'

Callback Queue (Task Queue)

  • Là hàng đợi FIFO (First In First Out)
  • Chứa các callback đã sẵn sàng để được thực thi
  • Còn gọi là MacroTask Queue

2.3. Microtask Queue (Job Queue)

Đặc điểm quan trọng:

  • Dành riêng cho Promises (.then(), .catch(), .finally())
  • Có độ ưu tiên CAO HƠN Callback Queue
  • Event loop sẽ xử lý toàn bộ Microtask Queue trước khi xử lý một macro task

Ví dụ so sánh độ ưu tiên:

javascript
console.log('1'); // Call Stack

setTimeout(() => {
    console.log('2'); // Macro Task
}, 0);

Promise.resolve().then(() => {
    console.log('3'); // Microtask
});

console.log('4'); // Call Stack

// Output: 1, 4, 3, 2
// Giải thích:
// - 1 và 4 chạy đồng bộ
// - Promise.then vào Microtask Queue (ưu tiên)
// - setTimeout vào Macro Task Queue
// - Xử lý hết Microtask (3) trước khi xử lý Macro task (2)

Microtask có thể tạo microtask khác:

javascript
Promise.resolve().then(() => {
    console.log('Microtask 1');
    Promise.resolve().then(() => {
        console.log('Microtask 2');
    });
}).then(() => {
    console.log('Microtask 3');
});

// Output: Microtask 1, Microtask 2, Microtask 3
// Giải thích: Xử lý hết tất cả microtask trong queue trước khi chuyển sang macro task

2.4. Vai trò của Event Loop

Event Loop là gì?

Event Loop là một vòng lặp vô hạn, hoạt động như một điều phối viên, liên tục kiểm tra:

  • Call Stack có trống không?
  • Microtask Queue có task không?
  • Macro Task Queue có task không?

Thuật toán chi tiết:

text
while (true) {
    // 1. Kiểm tra Call Stack
    if (callStack.isEmpty()) {
        // 2. Xử lý Microtask Queue (ưu tiên cao nhất)
        while (microTaskQueue.isNotEmpty()) {
            task = microTaskQueue.dequeue();
            callStack.push(task);
            execute(task);
            callStack.pop();
        }
        
        // 3. Xử lý một Macro Task (chỉ một task duy nhất)
        if (macroTaskQueue.isNotEmpty()) {
            task = macroTaskQueue.dequeue();
            callStack.push(task);
            execute(task);
            callStack.pop();
        }
        
        // 4. Render UI (nếu cần và đủ thời gian)
        renderUI();
    }
}

Bài tập phân tích chi tiết:

javascript
console.log('Start');

setTimeout(() => {
    console.log('Timeout 1');
    Promise.resolve().then(() => {
        console.log('Promise 1');
    });
}, 0);

Promise.resolve().then(() => {
    console.log('Promise 2');
    setTimeout(() => {
        console.log('Timeout 2');
    }, 0);
});

console.log('End');

// Hãy phân tích từng bước:
/*
Bước 1: Chạy đồng bộ
- console.log('Start') → in 'Start'
- setTimeout(0) → gửi callback1 vào Macro Task Queue
- Promise.resolve().then(...) → gửi callback Promise2 vào Microtask Queue
- console.log('End') → in 'End'

Bước 2: Call Stack rỗng, xử lý Microtask Queue
- Lấy callback Promise2: in 'Promise 2'
- Trong callback, gặp setTimeout → gửi callback Timeout2 vào Macro Task Queue

Bước 3: Microtask Queue rỗng, xử lý Macro Task Queue
- Lấy callback Timeout1: in 'Timeout 1'
- Trong callback, gặp Promise.resolve().then(...) → gửi callback Promise1 vào Microtask Queue

Bước 4: Kiểm tra lại Microtask Queue
- Lấy callback Promise1: in 'Promise 1'

Bước 5: Microtask Queue rỗng, xử lý Macro Task tiếp theo
- Lấy callback Timeout2: in 'Timeout 2'

Output: Start, End, Promise 2, Timeout 1, Promise 1, Timeout 2
*/

PHẦN 3: XỬ LÝ BẤT ĐỒNG BỘ THỜI KỲ ĐẦU - CALLBACKS

3.1. Định nghĩa Callback function

Callback là gì?

Callback là một hàm được truyền như tham số vào hàm khác và sẽ được gọi lại sau đó.

Phân loại callback:

Callback đồng bộ (Synchronous Callback):
javascript
function greet(name, callback) {
    console.log(`Xin chào ${name}`);
    callback(); // Gọi ngay lập tức
}

function sayGoodbye() {
    console.log('Tạm biệt!');
}

greet('John', sayGoodbye);
// Output:
// Xin chào John
// Tạm biệt!
Callback bất đồng bộ (Asynchronous Callback):
javascript
function fetchData(callback) {
    // Mô phỏng gọi API mất 2 giây
    setTimeout(() => {
        const data = { id: 1, name: 'John Doe' };
        callback(data);
    }, 2000);
}

function processData(data) {
    console.log('Dữ liệu nhận được:', data);
}

console.log('Bắt đầu lấy dữ liệu...');
fetchData(processData);
console.log('Câu lệnh này chạy ngay, không đợi dữ liệu');

// Output:
// Bắt đầu lấy dữ liệu...
// Câu lệnh này chạy ngay, không đợi dữ liệu
// (2 giây sau)
// Dữ liệu nhận được: { id: 1, name: 'John Doe' }

3.3. Hạn chế: Callback Hell (Pyramid of Doom)

Ví dụ thực tế: Xây dựng hệ thống đặt vé máy bay

javascript
// Giả sử các hàm API
function getUser(userId, callback) {
    setTimeout(() => {
        callback(null, { id: userId, name: 'John', membership: 'gold' });
    }, 1000);
}

function getFlights(user, callback) {
    setTimeout(() => {
        callback(null, [
            { id: 'FL001', from: 'Hanoi', to: 'Saigon', price: 200 },
            { id: 'FL002', from: 'Hanoi', to: 'Danang', price: 150 }
        ]);
    }, 1000);
}

function checkPromotion(membership, flight, callback) {
    setTimeout(() => {
        const discount = membership === 'gold' ? 0.2 : 0;
        callback(null, { ...flight, discount, finalPrice: flight.price * (1 - discount) });
    }, 500);
}

function bookTicket(flight, callback) {
    setTimeout(() => {
        callback(null, { bookingId: 'BK123', flight, status: 'confirmed' });
    }, 1000);
}

function processPayment(booking, callback) {
    setTimeout(() => {
        callback(null, { ...booking, paymentStatus: 'paid' });
    }, 1000);
}

function sendConfirmation(booking, callback) {
    setTimeout(() => {
        callback(null, { ...booking, emailSent: true });
    }, 500);
}

// CALLBACK HELL - 6 cấp độ lồng nhau!
function bookFlightForUser(userId) {
    getUser(userId, (err, user) => {
        if (err) {
            console.error('Lỗi lấy user');
            return;
        }
        
        getFlights(user, (err, flights) => {
            if (err) {
                console.error('Lỗi lấy chuyến bay');
                return;
            }
            
            // Chọn chuyến bay đầu tiên
            checkPromotion(user.membership, flights[0], (err, flightWithDiscount) => {
                if (err) {
                    console.error('Lỗi kiểm tra khuyến mãi');
                    return;
                }
                
                bookTicket(flightWithDiscount, (err, booking) => {
                    if (err) {
                        console.error('Lỗi đặt vé');
                        return;
                    }
                    
                    processPayment(booking, (err, paidBooking) => {
                        if (err) {
                            console.error('Lỗi thanh toán');
                            return;
                        }
                        
                        sendConfirmation(paidBooking, (err, finalBooking) => {
                            if (err) {
                                console.error('Lỗi gửi email');
                                return;
                            }
                            
                            console.log('Đặt vé thành công:', finalBooking);
                        });
                    });
                });
            });
        });
    });
}

bookFlightForUser(1);

Vấn đề của Callback Hell:

  • Khó đọc: Code hình tam giác ngược
  • Khó bảo trì: Thêm bước mới rất phức tạp
  • Khó xử lý lỗi: Phải kiểm tra lỗi ở mọi cấp độ
  • Khó tái sử dụng: Logic bị gắn chặt vào cấu trúc
  • Khó debug: Không biết lỗi xảy ra ở đâu

PHẦN 4: PROMISES

4.1. Định nghĩa Promise

Promise là gì?

Promise là một object đại diện cho một giá trị chưa tồn tại tại thời điểm hiện tại nhưng sẽ có trong tương lai.

So sánh với thực tế:

Giống như hóa đơn đặt hàng: Bạn đặt hàng (tạo promise), bạn chưa có hàng ngay (pending), sau đó hoặc nhận được hàng (fulfilled) hoặc được báo hết hàng (rejected).

4.2. Các trạng thái của Promise

text
[Pending]
    |
    |--> [Fulfilled] (Resolved) - Thành công, có kết quả
    |
    |--> [Rejected] - Thất bại, có lý do

Quy tắc bất biến:

  • Pending -> Fulfilled: Một chiều, không quay lại
  • Pending -> Rejected: Một chiều, không quay lại
  • Không thể từ Fulfilled -> Rejected hay ngược lại

4.3. Tạo Promise

Cú pháp cơ bản:

javascript
const promise = new Promise((resolve, reject) => {
    // Thực hiện tác vụ bất đồng bộ
    
    if (/* thành công */) {
        resolve(giá_trị); // Chuyển sang trạng thái fulfilled
    } else {
        reject(lỗi); // Chuyển sang trạng thái rejected
    }
});

Ví dụ chi tiết:

javascript
function downloadFile(url) {
    return new Promise((resolve, reject) => {
        console.log(`Bắt đầu tải file từ ${url}...`);
        
        setTimeout(() => {
            const success = Math.random() > 0.3; // 70% thành công
            
            if (success) {
                const file = {
                    name: url.split('/').pop(),
                    size: Math.floor(Math.random() * 1000) + 'KB',
                    content: 'Nội dung file...'
                };
                resolve(file); // Thành công
            } else {
                reject(new Error(`Không thể tải file từ ${url}`)); // Thất bại
            }
        }, 2000);
    });
}

const downloadPromise = downloadFile('https://example.com/image.jpg');
console.log(downloadPromise); // Promise {}

4.4. Tiêu thụ Promise: .then(), .catch(), .finally()

.then() - Xử lý khi thành công

javascript
downloadFile('https://example.com/image.jpg')
    .then((file) => {
        console.log('Tải file thành công!');
        console.log('Thông tin file:', file);
        return file; // Có thể return để chain
    });

.catch() - Xử lý khi thất bại

javascript
downloadFile('https://example.com/image.jpg')
    .then((file) => {
        console.log('File:', file);
    })
    .catch((error) => {
        console.error('Có lỗi:', error.message);
        console.error('Thời gian:', new Date().toLocaleString());
    });

.finally() - Luôn chạy dù thành công hay thất bại

javascript
function showLoadingSpinner() {
    console.log('🔄 Đang tải...');
}

function hideLoadingSpinner() {
    console.log('✅ Hoàn tất!');
}

showLoadingSpinner();

downloadFile('https://example.com/image.jpg')
    .then((file) => {
        console.log('📁 File:', file.name);
    })
    .catch((error) => {
        console.error('❌', error.message);
    })
    .finally(() => {
        hideLoadingSpinner(); // Luôn chạy
    });

4.5. Chaining Promises

Nguyên tắc cốt lõi:

Mỗi .then() trả về một Promise mới, cho phép chain.

javascript
function getUser(id) {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve({ id, name: 'John Doe', email: 'john@example.com' });
        }, 1000);
    });
}

function getPosts(userId) {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve([
                { id: 1, title: 'Post 1', userId },
                { id: 2, title: 'Post 2', userId }
            ]);
        }, 1000);
    });
}

function getComments(postId) {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve([
                { id: 1, text: 'Great post!', postId },
                { id: 2, text: 'Thanks for sharing', postId }
            ]);
        }, 500);
    });
}

// CHAINING - Code phẳng, dễ đọc
getUser(1)
    .then(user => {
        console.log('User:', user);
        return getPosts(user.id); // Trả về promise
    })
    .then(posts => {
        console.log('Posts:', posts);
        return getComments(posts[0].id); // Trả về promise
    })
    .then(comments => {
        console.log('Comments:', comments);
        console.log('Hoàn thành!');
    })
    .catch(error => {
        console.error('Lỗi ở bất kỳ đâu trong chain:', error);
    });

Các kiểu return trong .then():

javascript
Promise.resolve(1)
    .then(value => {
        console.log('Step 1:', value); // 1
        return value * 2; // Trả về giá trị thường -> Promise.resolve(2)
    })
    .then(value => {
        console.log('Step 2:', value); // 2
        return Promise.resolve(value * 3); // Trả về promise -> promise này resolve 6
    })
    .then(value => {
        console.log('Step 3:', value); // 6
        return Promise.reject(new Error('Lỗi cố ý')); // Trả về promise reject
    })
    .catch(error => {
        console.log('Bắt lỗi:', error.message);
        return 10; // Promise.resolve(10) - recovery
    })
    .then(value => {
        console.log('Step 4:', value); // 10 (vẫn chạy sau catch)
    });

4.6. Các phương thức tĩnh hữu ích

Promise.all() - Chờ tất cả hoàn thành

javascript
const promises = [
    downloadFile('https://example.com/file1.jpg'),
    downloadFile('https://example.com/file2.jpg'),
    downloadFile('https://example.com/file3.jpg')
];

Promise.all(promises)
    .then(files => {
        console.log('Tất cả files đã tải xong:');
        files.forEach((file, index) => {
            console.log(`${index + 1}. ${file.name} (${file.size})`);
        });
    })
    .catch(error => {
        console.error('Một file bị lỗi, dừng tất cả:', error.message);
    });

// Lưu ý: Promise.all thất bại nhanh (fail-fast)
// Nếu một promise reject, nó reject ngay lập tức

Promise.race() - Thằng nào nhanh nhất

javascript
const slow = new Promise(resolve => {
    setTimeout(() => resolve('Chậm'), 3000);
});

const fast = new Promise(resolve => {
    setTimeout(() => resolve('Nhanh'), 1000);
});

const withTimeout = new Promise((_, reject) => {
    setTimeout(() => reject(new Error('Timeout')), 2000);
});

Promise.race([slow, fast])
    .then(result => console.log('Winner:', result)); // 'Nhanh'

Promise.race([slow, withTimeout])
    .then(result => console.log('Success:', result))
    .catch(error => console.log('Error:', error.message)); // 'Error: Timeout'

Promise.allSettled() - Chờ tất cả, không quan tâm kết quả

javascript
const promises = [
    Promise.resolve(1),
    Promise.reject(new Error('Lỗi 2')),
    new Promise(resolve => setTimeout(() => resolve(3), 1000))
];

Promise.allSettled(promises)
    .then(results => {
        results.forEach((result, index) => {
            if (result.status === 'fulfilled') {
                console.log(`Promise ${index + 1}: thành công với giá trị ${result.value}`);
            } else {
                console.log(`Promise ${index + 1}: thất bại với lỗi ${result.reason.message}`);
            }
        });
    });

// Output:
// Promise 1: thành công với giá trị 1
// Promise 2: thất bại với lỗi Lỗi 2
// Promise 3: thành công với giá trị 3

Promise.any() - Thằng thành công đầu tiên

javascript
const p1 = Promise.reject(new Error('Lỗi 1'));
const p2 = new Promise(resolve => setTimeout(() => resolve('Thành công 2'), 500));
const p3 = new Promise(resolve => setTimeout(() => resolve('Thành công 3'), 1000));

Promise.any([p1, p2, p3])
    .then(result => {
        console.log('Thành công đầu tiên:', result); // 'Thành công 2' sau 500ms
    })
    .catch(error => {
        console.log('Tất cả đều thất bại:', error.errors);
    });

// Nếu tất cả đều thất bại
Promise.any([
    Promise.reject(new Error('Lỗi A')),
    Promise.reject(new Error('Lỗi B'))
]).catch(error => {
    console.log(error instanceof AggregateError); // true
    console.log(error.errors); // [Error: Lỗi A, Error: Lỗi B]
});

PHẦN 5: ASYNC/AWAIT

5.1. Từ khóa async

Định nghĩa:

Khi thêm async trước một hàm, hàm đó sẽ luôn trả về một Promise.

javascript
// Các cách viết tương đương
async function foo1() {
    return 42;
}
// Tương đương với:
function foo2() {
    return Promise.resolve(42);
}

async function bar1() {
    throw new Error('Lỗi');
}
// Tương đương với:
function bar2() {
    return Promise.reject(new Error('Lỗi'));
}

// Kiểm tra
foo1().then(console.log); // 42
bar1().catch(err => console.log(err.message)); // Lỗi

5.2. Từ khóa await

Định nghĩa:

  • await chỉ được dùng trong hàm async
  • Nó tạm dừng execution của hàm async cho đến khi Promise được resolve/reject
  • Trả về kết quả của Promise (nếu resolve) hoặc throw lỗi (nếu reject)
javascript
function delay(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
}

async function demonstrateAwait() {
    console.log('Bắt đầu');
    
    await delay(2000); // Tạm dừng 2 giây
    console.log('Sau 2 giây');
    
    await delay(1000); // Tạm dừng thêm 1 giây
    console.log('Sau 3 giây');
    
    return 'Hoàn thành';
}

demonstrateAwait().then(result => console.log(result));

So sánh Promise chain vs Async/Await:

Promise Chain
function getUserDataPromise(userId) {
    return new Promise((resolve, reject) => {


        // Giả lập lấy user
        setTimeout(() => {
            const user = { id: userId, name: "Manh" };


            // Giả lập lấy posts
            setTimeout(() => {
                const posts = ["Post 1", "Post 2"];


                resolve({ user, posts });


            }, 1000);


        }, 1000);


    })
    .catch(error => console.error(error));
}


📌 Cách dùng
getUserDataPromise(1)
    .then(data => console.log(data));
Async/Await
async function getUserDataAsync(userId) {
    try {


        // Chờ lấy user
        const user = await new Promise((resolve) => {
            setTimeout(() => {
                resolve({ id: userId, name: "Manh" });
            }, 1000);
        });


        // Chờ lấy posts
        const posts = await new Promise((resolve) => {
            setTimeout(() => {
                resolve(["Post 1", "Post 2"]);
            }, 1000);
        });


        return { user, posts };


    } catch (error) {
        console.error(error);
    }
}


📌 Cách dùng
getUserDataAsync(1)
    .then(data => console.log(data));

5.3. Xử lý lỗi với try...catch

javascript
async function fetchUserData(userId) {
    try {


        const user = await getUser(userId);
        console.log("User:", user);


        return user;


    } catch (error) {


        console.error("Không thể lấy user:", error);


        return {
            id: userId,
            name: "Unknown",
            error: true
        };
    }
}
